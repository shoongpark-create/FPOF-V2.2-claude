# 수요 예측 & 가격 최적화

> 주간 판매 데이터를 분석하여 재고 소진 예측, 리오더 타이밍, 가격 탄력성, BTA 리밸런싱,
> SPOT 기획 후보까지 산출하는 QR 의사결정 엔진.
> 경영목표 #3 "QR 비중 60%" 달성의 핵심 스킬.

## 기존 스킬과의 관계

| 기존 스킬 | 역할 | demand-optimizer에서의 위치 |
|-----------|------|--------------------------|
| `sales-analysis` | 매출/KPI 분석 (정기 리뷰) | Step 1~2 판매 데이터 소스, KPI 프레임워크 참조 |
| `qr-process` | 리오더/SPOT 실행 프로세스 | Step 4~5 결과를 qr-process에 전달 — 의사결정→실행 |
| `musinsa-ranking` | 외부 시장 판매 동향 | Step 3 경쟁 시그널 — 외부 수요 교차 검증 |
| `trend-radar` | 트렌드 시그널 정량화 | Step 5 SPOT 후보 선정 시 트렌드 적합도 참조 |

**demand-optimizer = sales-analysis의 "예측 + 최적화" 확장판**

`sales-analysis`는 과거를 분석하고, `demand-optimizer`는 미래를 예측한다.
예측 결과는 `qr-process`의 리오더/SPOT 의뢰서로 직접 연결된다.

## 언제 사용

- "이 상품 리오더 해야 해?" — 소진율 예측 기반 판단
- "QR 대상 뽑아줘" — 전 스타일 스캔 후 리오더/SPOT 후보 자동 추출
- "할인 언제 시작해?" — 가격 탄력성 분석 기반 타이밍 제안
- "BTA 조정해야 해?" — 실적 기반 포트폴리오 리밸런싱
- Check 단계에서 주간/격주 정기 리뷰 시
- 인시즌 중 판매 급등/급락 아이템 감지 시

### Step 1: 판매 속도 분석 (Sell-Through Velocity)

대시보드 JSON 데이터를 로드하여 스타일별 판매 속도를 계산한다.

#### 핵심 지표 계산

| 지표 | 계산식 | 의미 |
|------|--------|------|
| **판매율 (STR)** | 판매수량 / 입고수량 × 100 | 재고 소화 비율 |
| **일간 판매속도 (DRR)** | 기간 판매수량 / 판매일수 | 하루 평균 판매량 |
| **주간 판매속도 (WRR)** | DRR × 7 | 주간 판매 추세 |
| **판매속도 변화율** | (금주 WRR - 전주 WRR) / 전주 WRR | 가속/감속 판단 |
| **잔여 재고일수** | 잔여재고 / DRR | 현재 속도로 며칠 후 소진 |
| **순매출 기여율** | STY 순매출 / 전체 순매출 × 100 | 매출 기여도 |

#### 스타일별 판매 속도 테이블

```markdown
| STY | 상품명 | 입고 | 판매 | 재고 | STR% | DRR | 속도변화 | 잔여일 | 등급 |
|-----|--------|------|------|------|------|-----|---------|--------|------|
```

#### 판매 등급 분류

| 등급 | STR 기준 | DRR 기준 | 의미 | 권장 액션 |
|------|---------|---------|------|----------|
| **S** (히트) | ≥80% | 상위 10% | 폭발적 판매 | 즉시 리오더 |
| **A** (호조) | 60~79% | 상위 30% | 안정적 판매 | 리오더 검토 |
| **B** (보통) | 40~59% | 중간 | 정상 소화 | 모니터링 |
| **C** (부진) | 20~39% | 하위 30% | 소화 느림 | 가격 조정 검토 |
| **D** (체류) | <20% | 하위 10% | 재고 적체 | 할인/이동/소진 전략 |

### Step 2: 재고 소진 예측 (Inventory Depletion Forecast)

현재 판매 속도 기반으로 재고 소진 시점을 예측한다.

#### 소진 예측 모델

```
예상 소진일 = 현재일 + (잔여재고 / DRR)

시나리오 분석:
  낙관: DRR × 1.2 (판매 가속 가정)
  기본: DRR × 1.0 (현재 속도 유지)
  보수: DRR × 0.8 (판매 감속 가정)
```

#### 소진 예측 테이블

```markdown
| STY | 상품명 | 잔여재고 | DRR | 소진(낙관) | 소진(기본) | 소진(보수) | 시즌 잔여 | 판단 |
|-----|--------|---------|-----|----------|----------|----------|----------|------|
```

**판단 기준:**
| 상황 | 의미 | 액션 |
|------|------|------|
| 소진일 < 시즌 종료 - 리드타임 | 리오더 윈도우 열림 | 리오더 발주 |
| 소진일 ≈ 시즌 종료 | 적정 재고 | 모니터링 |
| 소진일 > 시즌 종료 + 30일 | 재고 과잉 리스크 | 할인/이동 검토 |
| 소진일 이미 경과 (품절) | 기회 손실 발생 | 긴급 리오더/SPOT |

---

### Step 3: 리오더 타이밍 최적화 (Reorder Timing)

리드타임을 역산하여 최적 발주 시점을 계산한다.

#### 리오더 의사결정 매트릭스

```
리오더 발동 조건:
  ① 판매등급 S 또는 A
  ② 잔여 재고일 ≤ 리드타임 + 안전재고일(7일)
  ③ 시즌 잔여기간 > 리드타임 × 2 (리오더 후 판매 기간 확보)
  ④ 판매속도 변화율 ≥ 0 (감속 중이면 물량 보수적으로)
```

#### 리오더 물량 산정

```
기본 물량 = DRR × 시즌 잔여일수 × 0.7 (안전 계수)
보정:
  - 속도 가속 중(변화율 > +10%): × 1.2
  - 속도 안정(변화율 ±10%): × 1.0
  - 속도 감속 중(변화율 < -10%): × 0.7
  - 사이즈별 판매 비율 반영 (편중 사이즈 추가 배분)
```

#### 리오더 알림 리스트

```markdown
## 리오더 알림

### 🔴 긴급 (즉시 발주)
| STY | 상품명 | 잔여재고 | 소진예측 | 권장물량 | 예상비용 | 발주기한 |
|-----|--------|---------|---------|---------|---------|---------|

### 🟡 정상 (1~2주 내 발주)
| STY | 상품명 | 잔여재고 | 소진예측 | 권장물량 | 예상비용 | 발주기한 |
|-----|--------|---------|---------|---------|---------|---------|

### 🟢 대기 (모니터링)
| STY | 상품명 | 잔여재고 | 소진예측 | 재검토일 |
|-----|--------|---------|---------|---------|
```

**qr-process 연동**: 긴급/정상 리스트의 각 아이템은 `qr-process` 스킬의 리오더 의뢰서로 직접 전환 가능.

### Step 4: 가격 탄력성 분석 (Price Elasticity)

할인 시점 및 할인율 최적화를 위한 분석.

#### 할인 대상 판별 (판매등급 C, D 아이템)

```markdown
| STY | 상품명 | 현재가 | 판매율 | 잔여재고 | 재고일수 | 재고금액 | 할인 권고 |
|-----|--------|--------|--------|---------|---------|---------|----------|
```

#### 할인율 시뮬레이션

| 할인율 | 예상 판매속도 배수 | 예상 추가 판매량 | 예상 순매출 | 재고 소진일 변화 |
|--------|------------------|----------------|-----------|----------------|
| 10% | ×1.3 | | | |
| 20% | ×1.8 | | | |
| 30% | ×2.5 | | | |
| 40%+ | ×3.0+ | | | |

**할인 판매속도 배수 기준:**
- 카테고리별 과거 할인 반응 데이터가 있으면 실데이터 우선
- 없으면 업계 평균 탄력성 적용 (위 표 기준)

#### 할인 타이밍 권고

```
최적 할인 시점 판단:
  ① 시즌 잔여 50% 지점에서 C등급 → 10~20% 할인 시작
  ② 시즌 잔여 30% 지점에서 D등급 → 30% 할인
  ③ 시즌 종료 2주 전 → 잔여 재고 소진 할인 (최대 40~50%)
  
주의: 
  - 히트상품(S/A등급) 절대 할인 금지 — 브랜드 가치 훼손
  - Counter Metric 감시: 전체 할인 매출 비중 > 30% 시 경고
```

### Step 5: SPOT 기획 후보 추천

히트상품 변형 + 트렌드 반응형 신규 아이템을 SPOT 생산 후보로 추천.

#### SPOT 후보 선정 기준

| 기준 | 조건 | 가중치 |
|------|------|--------|
| 원본 판매 성과 | S/A등급 히트상품의 변형 | 0.30 |
| 트렌드 적합도 | trend-radar 스코어 A등급 이상 | 0.25 |
| 생산 실현성 | 기존 소재/공장 활용 가능 | 0.25 |
| 시즌 잔여 | 리드타임 후 최소 4주 판매 가능 | 0.20 |

#### SPOT 후보 리스트

```markdown
| # | SPOT 아이디어 | 원본 STY | 변형 내용 | 트렌드 연결 | 예상 리드타임 | SPOT 스코어 |
|---|-------------|---------|----------|-----------|-------------|------------|
| 1 | [히트 아이템 컬러 추가] | [원본] | 컬러웨이 +2 | [트렌드] | 3주 | /10 |
| 2 | [히트 아이템 소재 변형] | [원본] | 소재 변경 | [트렌드] | 5주 | /10 |
| 3 | [트렌드 반응형 신규] | 신규 | [디자인] | [트렌드] | 6주 | /10 |
```

**qr-process 연동**: 승인된 SPOT 후보는 `qr-process` 스킬의 SPOT 의뢰서로 전환.

### Step 6: BTA 포트폴리오 리밸런싱

실적 기반으로 Basic/Trend/Accent 배분을 재조정한다.

#### 현행 BTA 실적 분석

```markdown
| BTA | 기획 STY수 | 기획 비중 | 실적 매출 | 실적 비중 | 평균 STR | 판단 |
|-----|-----------|----------|---------|----------|---------|------|
| Basic | | % | | % | % | |
| Trend | | % | | % | % | |
| Accent | | % | | % | % | |
```

#### BTA 이동 권고

```
BTA 이동 판단 기준:

B → T 이동: Basic 아이템이 예상 외 히트 → Trend로 승격, 마케팅 투자 확대
T → B 이동: Trend 아이템이 안정적 반복 판매 → Basic으로 전환, 리오더 대상
T → A 이동: Trend 아이템이 부진 → Accent로 변경, 실험적 포지셔닝
A → T 이동: Accent 아이템이 예상 외 반응 → Trend로 승격, 물량 확대

주의:
  - bta-guideline.md 기준 준수 (Basic 안정 매출, Trend 시즌 견인, Accent 미래 투자)
  - 리밸런싱은 시즌 중간(50% 지점)과 시즌 후반(75% 지점)에 실행
```

#### BTA 리밸런싱 권고 테이블

```markdown
| STY | 상품명 | 현재 BTA | 권고 BTA | 근거 | 권고 액션 |
|-----|--------|---------|---------|------|----------|
```
