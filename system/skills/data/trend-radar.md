# 트렌드 레이더

> 멀티 소스 트렌드 시그널을 자동 수집하고 5축 스코어링으로 정량화하여,
> 와키윌리가 이번 시즌에 어떤 트렌드에 베팅할지 데이터 기반으로 판단하는 레이더 시스템.

## 기존 스킬과의 관계

| 기존 스킬 | 역할 | trend-radar에서의 위치 |
|-----------|------|----------------------|
| `trend-research` | 수동 리서치, 정성 분석 | Step 2 글로벌 시그널 + Step 5 브랜드 필터의 기반 |
| `market-intelligence` | 네이버/구글 키워드 트렌드 | Step 1 서치 시그널 데이터 피드 |
| `musinsa-ranking` | 무신사 판매 랭킹 | Step 3 리테일 시그널 데이터 피드 |
| `musinsa-release` | 무신사 신상품 발매 | Step 3 리테일 시그널 데이터 피드 |
| `pinterest-crawl` | 비주얼 레퍼런스 수집 | Step 4 비주얼 시그널 데이터 피드 |

**trend-radar = 기존 5개 스킬의 결과물을 통합 + 스코어링 + BTA 매핑**

기존 스킬을 대체하지 않는다. 각 스킬이 수집한 원천 데이터 위에 분석 레이어를 얹는 **오케스트레이션 스킬**이다.

## 언제 사용

- "요즘 뭐가 뜨고 있어?" — 감이 아닌 숫자로 대답해야 할 때
- "이번 시즌 어떤 트렌드에 집중해야 해?" — BTA 배분 의사결정이 필요할 때
- "트렌드 카드 만들어줘" — 시즌 기획 킥오프 전 트렌드 맵 필요할 때
- PDCA Plan 단계에서 `trend-research` 전 또는 후에 데이터 기반 보강
- 시즌 중간에 트렌드 변동을 정량적으로 추적할 때

## 사전 준비

1. `.fpof-state.json` → 현재 시즌 확인
2. `presets/wacky-willy/brand.config.json` → 브랜드 DNA, 전략 방향
3. `presets/wacky-willy/categories.json` → 카테고리 구조, 히어로 아이템
4. `presets/wacky-willy/personas.json` → 코어 타겟 프로파일
5. `presets/wacky-willy/visual-identity.json` → 비주얼 키워드 (스코어링 기준)
6. 기존 데이터 확인:
   - `workspace/musinsa-ranking/` → 최신 랭킹 데이터 존재 여부
   - `workspace/musinsa-release/` → 최신 발매 데이터 존재 여부
   - `config/market-intel-config.json` → 마켓 인텔 최신 리포트 존재 여부
   - `knowledge/` → 이전 시즌 트렌드 적중률 (있을 경우)

### Step 1: 서치 시그널 수집 (Search Signals)

소비자가 검색하는 키워드에서 트렌드 시그널을 읽는다.

**데이터 소스:**
- `market-intelligence` 스킬의 최신 리포트 (있으면 로드)
- 없으면: 웹서치로 네이버 트렌드, 구글 트렌드 주요 키워드 동향 수집

**수집 항목:**
| 항목 | 소스 | 추출 데이터 |
|------|------|-----------|
| 키워드 검색량 | 네이버 데이터랩 | 주간 검색 지수, WoW 변화율 |
| 관심도 추이 | 구글 트렌드 | 관심도 곡선, 라이징 쿼리 |
| 연관 검색어 | 네이버/구글 | 새로 부상하는 연관 키워드 |
| 연령 분포 | 네이버 데이터랩 | 20대 코어타겟 비중 |

**추출 결과 형태:**
```
서치 시그널 #1: [키워드]
- 검색 지수: [숫자] (WoW: +/-%)
- 20대 비중: [%]
- 라이징 쿼리: [연관 키워드 3~5개]
- 트렌드 곡선: [상승/정체/하락]
```

---

### Step 2: 글로벌 시그널 수집 (Global Signals)

글로벌 패션 씬에서 반복 등장하는 테마를 포착한다.

**수집 방법:**
- 웹서치로 최신 패션 위크 리포트, 스트릿 스냅 트렌드, SNS 바이럴 수집
- `trend-research` 스킬의 기존 산출물이 있으면 참조

**수집 항목:**
| 항목 | 검색 키워드 예시 | 추출 데이터 |
|------|----------------|-----------|
| 런웨이 트렌드 | "fashion week 2026 trends" | 반복 테마, 실루엣, 컬러 |
| 스트릿 스냅 | "street style trends 2026" | 실제 착용 트렌드, 스타일링 |
| TikTok/인스타 바이럴 | "TikTok fashion trend viral 2026" | 바이럴 스타일, 해시태그 규모 |
| K-패션 동향 | "K-fashion trend 2026" | 한국 시장 특화 흐름 |
| 서브컬처 | "subculture fashion kitsch Y2K" | 와키윌리 DNA 연결 트렌드 |

**추출 결과 형태:**
```
글로벌 시그널 #1: [트렌드명]
- 출처: [런웨이/스트릿/SNS]
- 규모: [언급량, 해시태그 수, 조회수]
- 핵심 요소: [실루엣, 소재, 컬러, 그래픽]
- 대표 브랜드/인플루언서: [참여자]
- 지속성 판단: [일시적 밈 / 성장 중 / 안정기 / 쇠퇴기]
```

### Step 3: 리테일 시그널 수집 (Retail Signals)

실제 판매 데이터에서 트렌드의 상업적 증거를 찾는다.

**데이터 소스:**
- `musinsa-ranking` 스킬 결과 (최신 랭킹 엑셀/JSON 로드)
- `musinsa-release` 스킬 결과 (최신 발매 데이터 로드)
- 자체 판매 데이터 (`workspace/[시즌]/dashboard/data_*.json` 로드)

**수집 항목:**
| 항목 | 소스 | 추출 데이터 |
|------|------|-----------|
| 카테고리 랭킹 변동 | 무신사 랭킹 | 순위 상승/하락 아이템, 신규 진입 |
| 발매 트렌드 | 무신사 발매 | 활발한 브랜드, 가격대, 카테고리 분포 |
| 경쟁사 동향 | 무신사 랭킹 | 유사 포지셔닝 브랜드 상위 아이템 |
| 자사 판매 신호 | 대시보드 JSON | 예상 외 히트/부진 아이템 |
| 가격대 변화 | 무신사 랭킹+발매 | 주류 가격대 이동, 할인 전략 패턴 |

**데이터 없는 경우 대응:**
- 무신사 데이터 없음 → "무신사 랭킹/발매 수집을 먼저 실행할까요?" 제안
- 자사 판매 데이터 없음 → 리테일 시그널은 무신사 데이터만으로 진행

**추출 결과 형태:**
```
리테일 시그널 #1: [카테고리/아이템 유형]
- 랭킹 동향: [순위 변화, 신규 진입 여부]
- 가격대: [주류 가격 레인지]
- 상위 브랜드: [브랜드 리스트]
- 발매 활동: [최근 발매 건수, 품절 속도]
- 상업적 검증: [실제 판매로 증명된 수요]
```

### Step 4: 비주얼 시그널 수집 (Visual Signals)

비주얼 트렌드 방향성을 포착한다.

**데이터 소스:**
- `pinterest-crawl` 스킬 결과 (최근 수집 이미지 폴더 존재 여부 확인)
- 웹서치 기반 비주얼 트렌드 키워드

**수집 항목:**
| 항목 | 방법 | 추출 데이터 |
|------|------|-----------|
| 컬러 트렌드 | 웹서치 "color trend 2026 fashion" | 시즌 컬러, PANTONE COTY |
| 실루엣 트렌드 | 웹서치 "silhouette trend fashion" | 핏 방향 (오버사이즈/슬림/크롭 등) |
| 그래픽 트렌드 | 웹서치 "graphic design trend streetwear" | 프린트/그래픽 스타일 방향 |
| 소재 트렌드 | 웹서치 "fabric material trend fashion" | 주목 소재, 기능성 소재 |
| Pinterest 무드 | 기존 수집 이미지 분석 | 반복 등장 비주얼 요소 |

**와키윌리 비주얼 필터** (`visual-identity.json` 기준):
- 디자인 키워드 매칭: Doodle, Graffiti, Pop Art, Bold Lines, Vivid Colors
- 시그니처 컬러 연결성: #FEF200 (Yellow), Black, White, #68A8DB (Sky Blue)
- IP 캐릭터 접목 가능성: KIKY, 릴리, 프렌즈 캐릭터와의 결합 여부

**추출 결과 형태:**
```
비주얼 시그널 #1: [비주얼 트렌드명]
- 유형: [컬러/실루엣/그래픽/소재]
- 내용: [구체적 트렌드 설명]
- 브랜드 적합도: [visual-identity.json 키워드 매칭 수]
- IP 접목 가능성: [높음/중간/낮음]
```

### Step 5: 5축 트렌드 스코어링 (Trend Scoring Matrix)

수집한 모든 시그널을 트렌드 단위로 통합하고, 5축 매트릭스로 정량 평가한다.

**스코어링 기준 (각 1~10점):**

| 축 | 기준 | 1점 | 5점 | 10점 |
|---|------|-----|-----|------|
| **Velocity (상승 속도)** | 시그널이 얼마나 빠르게 성장 중인가 | 정체/하락 | 완만한 상승 | 급격한 상승, 바이럴 폭발 |
| **Volume (시장 규모)** | 시장에서 얼마나 큰 규모인가 | 니치, 소수 | 중간 규모 | 대중적, 매스마켓 침투 |
| **Relevance (브랜드 적합도)** | 와키윌리 DNA와 얼마나 맞는가 | DNA 충돌 | 부분 적합 | 완벽 매칭 (Kitsch/IP/Street) |
| **Longevity (예상 수명)** | 이 트렌드가 얼마나 오래갈 것인가 | 1개월 미만 밈 | 1~2시즌 | 3시즌 이상 지속 |
| **Whitespace (경쟁 공백)** | 경쟁사가 아직 선점하지 않은 영역인가 | 레드오션 | 일부 경쟁 | 블루오션, 선점 기회 |

**스코어링 소스 매핑:**

| 축 | 서치 시그널 | 글로벌 시그널 | 리테일 시그널 | 비주얼 시그널 |
|---|-----------|-------------|-------------|-------------|
| Velocity | WoW 변화율, 라이징 쿼리 | 해시태그 성장률 | 랭킹 순위 변동 | — |
| Volume | 검색 지수 절대값 | 조회수, 언급량 | 랭킹 순위, 발매 건수 | — |
| Relevance | 20대 비중 | DNA 키워드 매칭 | 경쟁사 포지셔닝 | 비주얼 키워드 매칭 |
| Longevity | 트렌드 곡선 형태 | 지속성 판단 | 재발매 인기 데이터 | 런웨이→스트릿 전파 여부 |
| Whitespace | 경쟁사 키워드 부재 | 경쟁 브랜드 참여도 | 무신사 랭킹 공백 | — |

**종합 점수 계산:**
```
총점 = (Velocity × 0.20) + (Volume × 0.15) + (Relevance × 0.30) + (Longevity × 0.15) + (Whitespace × 0.20)
```

- **Relevance 가중치 최대 (0.30)**: 아무리 큰 트렌드도 브랜드에 안 맞으면 의미 없음
- **Velocity + Whitespace 합계 (0.40)**: 빠르게 성장 + 선점 기회 = 즉시 액션 대상
- **Volume + Longevity 합계 (0.30)**: 안정적 수요 기반 판단

**등급 분류:**
| 등급 | 점수 | 의미 | 권장 액션 |
|------|------|------|----------|
| **S** | 8.0+ | 즉시 베팅 | 이번 시즌 메인 테마 후보 |
| **A** | 6.5~7.9 | 적극 검토 | 트렌드/악센트 라인 기획 |
| **B** | 5.0~6.4 | 모니터링 | 다음 시즌 후보, 관찰 지속 |
| **C** | 3.5~4.9 | 보류 | 시그널 약함, 재평가 필요 |
| **D** | 3.5 미만 | 패스 | 브랜드 부적합 또는 쇠퇴 |

### Step 6: BTA 트렌드 매핑

스코어링 결과를 BTA 프레임워크에 배분한다.

**매핑 로직:**

```
                  Longevity 높음 + Volume 높음
                          │
                     ┌────┴────┐
                     │  Basic  │  ← 안정 수요, 3시즌+ 지속, 리오더 대상
                     │  (B)    │     재발매 인기 상품 유형
                     └────┬────┘
                          │
              Velocity 높음 + Volume 중~높음
                          │
                     ┌────┴────┐
                     │  Trend  │  ← 성장 중, 1~2시즌 메인, 히트상품 후보
                     │  (T)    │     시즌 매출 견인, IMC 집중 대상
                     └────┬────┘
                          │
             Whitespace 높음 + Relevance 높음
                          │
                     ┌────┴────┐
                     │ Accent  │  ← 선점 기회, 브랜드 DNA 강화
                     │  (A)    │     미래 수익원, 실험적 아이템
                     └─────────┘
```

**BTA 배분 기준:**
| BTA | 주요 판단 축 | 조건 | 카테고리 연결 |
|-----|------------|------|-------------|
| Basic | Longevity ≥ 7, Volume ≥ 6 | 안정적, 지속적 수요 | 베이직 리오더 아이템 |
| Trend | Velocity ≥ 7, Volume ≥ 5 | 빠른 성장, 상업성 증명 | 시즌 메인 기획 아이템 |
| Accent | Whitespace ≥ 7, Relevance ≥ 7 | 선점 기회 + 브랜드 적합 | 실험/IP 협업 아이템 |

트렌드 하나가 복수의 BTA에 걸칠 수 있다. 이 경우 가장 높은 축을 우선 배정하되 부차적 활용도 표기한다.

### Step 7: 트렌드 카드 생성

스코어링 결과를 카드 형식으로 시각화한다. 카드 순서는 종합 점수 내림차순.

**트렌드 카드 포맷:**
```markdown
---
### 🔹 트렌드 카드 #[순번]: [트렌드명]

**등급**: [S/A/B/C/D] | **종합**: [점수]/10 | **BTA**: [Basic/Trend/Accent]

| 축 | 점수 | 근거 |
|---|------|------|
| Velocity | /10 | [1줄 근거] |
| Volume | /10 | [1줄 근거] |
| Relevance | /10 | [1줄 근거] |
| Longevity | /10 | [1줄 근거] |
| Whitespace | /10 | [1줄 근거] |

**트렌드 설명**: [2~3문장 설명]

**시그널 근거**:
- 서치: [키워드, 검색량, 변화율]
- 글로벌: [출처, 규모]
- 리테일: [무신사 랭킹/발매 데이터]
- 비주얼: [컬러/실루엣/소재 키워드]

**와키윌리 적용 아이디어**:
- 카테고리: [UNI/W/ACC 중 어디에 적용]
- 아이템: [구체적 아이템 제안]
- IP 연결: [캐릭터 접목 방안]
- 타이밍: [시즌 내 출시 시점 제안]
---
```

### Step 8: 시즌 트렌드 맵 & 타임라인

전체 트렌드 카드를 시즌 맵으로 종합한다.

**시즌 트렌드 맵:**
```
트렌드 포지셔닝 매트릭스

        Volume 높음
            │
     ┌──────┼──────┐
     │ Mass │ Hit  │
     │Trend │Zone  │  ← 여기가 히트상품 영역 (Velocity+Volume 모두 높음)
     │      │      │
─────┼──────┼──────┼───── Velocity
     │      │      │
     │ Fade │ Rise │
     │ Out  │      │  ← 여기서 선점 (Velocity 높지만 Volume 아직)
     │      │      │
     └──────┼──────┘
            │
        Volume 낮음

[각 트렌드를 매트릭스 위에 배치하여 표로 작성]
```

**타임라인 예측:**
| 트렌드 | 현재 단계 | 피크 예측 | 잔여 수명 | 액션 타이밍 |
|--------|----------|----------|----------|-----------|
| [트렌드A] | 성장 초기 | 3~4개월 후 | 2시즌+ | 지금 기획 시작 |
| [트렌드B] | 성장 중기 | 1~2개월 후 | 1시즌 | QR/SPOT 대응 |
| [트렌드C] | 정점 부근 | 현재 | 0.5시즌 | SPOT 빠른 대응 |
