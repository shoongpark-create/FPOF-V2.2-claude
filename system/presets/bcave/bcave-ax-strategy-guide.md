## 1. 왜 AX인가

### 1-1. CEO 비전 — "판단하는 사람이 되어야 합니다"

대표이사는 이미 명확한 방향을 제시했습니다:

> "앞으로 업무 경쟁력은 두 가지에서 결정됩니다.
> 첫째, 자신의 업(業)에 대한 깊은 이해.
> 둘째, AI를 활용해 그 이해를 빠르게 실행으로 연결하는 능력."

> "앞으로 우리는 AI를 사용하는 조직이 아니라 AI를 기반으로 일하는 조직이 되어야 합니다."

이것은 단순한 도구 도입이 아닙니다. **업무 방식의 근본적 전환**을 의미합니다.

### 1-2. 패션 산업이 직면한 AI 전환의 시급성

| 변화 동인 | 글로벌 현황 | 비케이브 영향 |
|----------|-----------|-------------|
| **수요 예측 AI** | Zara: AI 기반 재고 최적화로 폐기율 30% 감축, 소량 다품종 생산 체계 확립 | 베이직 초두 축소 + 리오더 설계의 정밀도 향상 가능 |
| **트렌드 분석** | Heuritech: 일 수백만 장 SNS 이미지 분석, 2,000+ 패션 속성 감지 | BTA 결정의 정량 근거 강화 (현재 정성 판단 의존) |
| **AI 촬영/비주얼** | Zara·H&M·Zalando: AI 생성 패션 포토그래피 도입, 촬영 비용 90% 절감 | 룩북·SNS 콘텐츠 생산 속도 10배 가속 |
| **SHEIN 모델** | SNS 트렌드 → 3일 내 디자인·생산, 일 수천 신상품 출시 | 속도 경쟁에서 뒤처질 위험 vs. 브랜드 가치 중심 차별화 |
| **국내 경쟁** | 삼성물산·LF·한섬 등 대기업 AI 전환 본격화, 스튜디오랩 '젠시' CES 최고혁신상 | 중견 패션사로서 선제적 AX가 경쟁 우위 확보의 마지막 기회 |

### 1-3. 왜 "행동 변화"인가 — AX ≠ 도구 도입

코르카 AX 팀 리드의 핵심 메시지:

> "AX의 본질은 AI 도구를 제공하는 것이 아니라, 개인·팀·조직이 동시에 행동을 변화시키는 것입니다.
> 도구만 제공해서는 실질적인 행동 변화나 지표 개선이 어렵습니다."

**비케이브에 적용하면:**

- **도구만 사는 것** = ChatGPT 라이선스 전사 배포 → 3개월 후 사용률 10% 미만
- **행동을 바꾸는 것** = "주간 리뷰 전에 AI가 데이터 분석을 먼저 하고, 사람은 전략적 판단에 집중"

패션 업무의 "코드 품질"에 해당하는 것은 **데이터 표준화, 프로세스 문서화, BTA 통합시트**입니다. 이것이 잘 정리되어야 AI가 정확한 결과를 낼 수 있습니다.

5. **문화적 저항** — "패션은 감(感)이야" 마인드셋

## 3. AX 프레임워크

### 비케이브 AX 3레이어 모델

```
┌─────────────────────────────────────────────┐
│           Layer 3. 조직 시스템 전환           │
│   데이터 기반 의사결정 · 프로세스 자동화      │
│   조직 플래트닝 · AI 거버넌스                │
├─────────────────────────────────────────────┤
│           Layer 2. 팀 워크플로우 변혁         │
│   P-M-D 가치사슬별 AI 에이전트화             │
│   BTA 통합시트 AI 연동 · 시즌 PDCA 자동화    │
├─────────────────────────────────────────────┤
│           Layer 1. 개인 역량 혁신             │
│   AI 리터러시 · 도파민 경험 · 챔피언 육성     │
│   "업(業) 이해 + AI 실행력" 결합             │
└─────────────────────────────────────────────┘
```

### Layer 1. 개인 — AI 리터러시 & 도파민 경험

**핵심 원칙:** "도구를 가르치지 말고, 문제를 해결하게 하라"

GS 그룹 사례에서 입증된 것처럼, AI를 교육 → 실습이 아닌 **문제 발견 → 즉시 해결 → 도파민**의 순서로 경험하게 해야 합니다.

#### 실행 프로그램

**1) 작업 로그 분석 (1인 1시간)**

각 팀원이 지난 1주일 업무를 시간·에너지 소모량과 함께 기록하고, AI를 활용해 "자동화 가능 영역"을 식별합니다.

```
예시 — 마케팅팀 김대리의 작업 로그:
- 경쟁사 SNS 모니터링 (주 3시간, 에너지 ★★☆) → AI 자동화 가능
- 주간 매출 리포트 작성 (주 2시간, 에너지 ★★★) → AI 자동화 가능
- 셀럽 섭외 미팅 (주 1시간, 에너지 ★☆☆) → 사람만 가능
- SNS 캡션 작성 (주 4시간, 에너지 ★★☆) → AI 초안 + 사람 검수
```

**2) 챔피언 발굴 & 육성 (부서당 1-2명)**

| 단계 | 내용 | 기간 |
|------|------|------|
| 발굴 | AI에 관심 있는 자발적 지원자 선발 (지원 의지 > 기술 역량) | 1주 |
| 집중 육성 | 챔피언 대상 1:1 짝 프롬프팅, 실무 문제 해결 경험 | 2-4주 |
| FOMO 확산 | 챔피언의 성과를 부서 내 공유, 동료들의 관심 유발 | 지속 |
| 팀 전파 | 챔피언이 부서 내 AI 사용 가이드 & 실전 사례 전파 | 지속 |

**3) AI 리터러시 3단계 교육**

| 레벨 | 대상 | 내용 | 목표 |
|------|------|------|------|
| **L1 — 이해** | 전 직원 | LLM 기본 개념, 프롬프팅 기초, 환각(hallucination) 이해 | AI가 뭘 할 수 있고 없는지 판단 |
| **L2 — 활용** | 실무자 | 업무별 프롬프트 라이브러리, FPOF 스킬 활용, 데이터 분석 | 일상 업무에 AI 통합 |
| **L3 — 설계** | 챔피언/리더 | AI 에이전트 설계, 워크플로우 자동화, 프롬프트 엔지니어링 | 부서 맞춤 AI 솔루션 구축 |

### Layer 2. 팀 — P-M-D 가치사슬 AI 에이전트화

**핵심 원칙:** "현장의 문제를 가장 잘 아는 사람이 AI를 만든다"

GS 그룹의 GS칼텍스 사례: 3,500명 중 2,000명이 2-3개월 만에 300개 AI 에이전트를 직접 제작.
비케이브 규모에 적용하면: **핵심 부서 10-15개 에이전트가 현실적 목표.**

#### 부서별 AI 에이전트 설계

| 부서 | 에이전트명 (가칭) | 기능 | 자동화 범위 |
|------|-----------------|------|-----------|
| **전략기획** | 마켓워치 | 경쟁사 모니터링, 트렌드 자동 수집, 시장 환경 변화 알림 | 리서치 시간 70% 절감 |
| **MD (상품기획)** | BTA플래너 | BTA 비율 시뮬레이션, SKU 배분 초안, 과거 실적 기반 추천 | 초안 작성 80% 자동화 |
| **디자인** | 캐드체커 | CAD맵 7대 항목 자동 점검, 3B 착장 위반 감지, 컬러 중복 검출 | 사전 점검 90% 자동화 |
| **마케팅** | 콘텐츠팩토리 | SNS 캡션 초안, B.A.M.P 자동 검증, 셀럽 체크리스트 보조 | 카피 초안 70% 자동화 |

| **경영지원** | 미니CEO | 주간 리뷰 자동 요약, 의사결정 필요 사안 하이라이트, 스케줄 관리 | 보고서 초안 80% 자동화 |

#### "패션 문제은행" 구축

GS의 문제은행 모델을 비케이브에 적용:

1. **Teams 채널** `#ax-문제은행` 개설
2. 누구나 "이거 AI로 해결할 수 없을까?" 형식으로 문제 등록
3. 주 1회 AX 챔피언 모임에서 해결 가능한 문제 선별
4. 챔피언이 2주 내 프로토타입 제작 → 현장 테스트
5. 성공 사례는 전사 공유 → FOMO 효과

### Layer 3. 조직 — 데이터 기반 의사결정 시스템

**핵심 원칙:** "기록이 바뀌면 평가가 바뀌고, 평가가 바뀌면 행동이 바뀐다"

#### 데이터 기록 혁신

현재의 문제:
- 매출 데이터는 있으나 분석이 수작업 (엑셀 기반)
- 트렌드 판단 근거가 기록되지 않음 ("감으로 했다")
- 회의 결정사항의 후속 추적이 느슨함

AI 시대의 기록 체계:

| 영역 | As-Is | To-Be |
|------|-------|-------|
| **매출 분석** | 엑셀 취합 → 수작업 리포트 | 자동 대시보드 + AI 인사이트 하이라이트 |
| **트렌드 근거** | 구두 공유, 개인 기억 의존 | AI 수집 → 구조화된 트렌드 브리프 |
| **회의록** | 메모장 or 미작성 | AI 자동 트랜스크립트 + 액션 아이템 추출 |
| **품평 결과** | 피드백이 구두로만 전달 | 체계적 기록 → BTA별 피드백 패턴 분석 |
| **고객 반응** | 리뷰 수작업 읽기 | AI 감성 분석 + 카테고리별 인사이트 |

#### 컨티뉴어스 피드백 도입

연 1-2회 시즌 리뷰 → **주간 AI 기반 피드백 사이클**로 전환:

```
[월] 주간 데이터 자동 수집·분석
[화] AI가 이슈·기회·위험 하이라이트 → 담당자 알림
[수] 팀별 15분 스탠드업 (AI 리포트 기반)
[목-금] 실행 & 기록
[일] 다음 주 자동 반복
```

#### 시나리오 A: BTA 비율 AI 시뮬레이션

**As-Is:**
```
MD가 과거 경험 + 판매 데이터를 엑셀로 취합
→ BTA 비율을 감(感)으로 설정
→ 품평에서 대표이사 피드백 후 조정

```

**To-Be:**
```
AI가 자동으로:
1) 과거 3시즌 BTA별 판매 실적·소진율·회전율 분석
2) 경쟁사 3사의 시즌 BTA 변화 추이 모니터링

4) 3가지 시나리오(보수적/균형/공격적) 시뮬레이션 제시

MD는:
- AI 추천안을 검토하고 브랜드 방향성에 맞게 조정
- "왜 이 비율인지" 데이터 근거를 품평 자료에 포함
- 대표이사에게 "감이 아닌 근거"로 설명 가능
```

**기대 효과:** BTA 비율 설정 시간 60% 단축, 의사결정 근거 투명화

| FPOF 적용 브랜드 수 | 2개 이상 | 시스템 확인 |
| 트렌드 리서치 정확도 | 베이스라인 대비 +20% | 사후 검증 |

#### Phase 3 (확산) — 비즈니스 임팩트

| 지표 | 목표 | 측정 방법 |

### 8-2. 거버넌스 구조

#### AX 추진 조직

```
[대표이사] — AX 스폰서, 방향 설정
    │
[AX 리드] — 전략기획실 (겸임 or 전담 1명)
    │        전사 AX 전략 수립, 진행 관리, 성과 보고
    │
[AX 챔피언 네트워크] — 부서별 1-2명 (10-15명)
    │        각 부서 AX 실행, 문제 발굴, 에이전트 개발
    │
[전 직원] — AI 리터러시 L1 이상, AI 활용 일상화
```

#### AI 사용 원칙

1. **데이터 보안**: 고객 개인정보, 미공개 매출 데이터는 외부 AI 서비스에 입력 금지
2. **브랜드 보이스**: 고객 대면 콘텐츠는 반드시 톤앤매너 프리셋 참조 후 사람이 최종 검수
3. **의사결정 책임**: AI는 보조, 최종 판단은 사람 (특히 셀럽, 가격, 수량 의사결정)
4. **투명성**: AI가 생성한 콘텐츠/분석은 "AI 보조" 표기, 근거 데이터 명시
5. **지속 개선**: 월 1회 AI 활용 사례 공유회, 분기 1회 AX 성과 리뷰

| **데이터 인프라 부족** | 높음 | 높음 | Phase 1에서 기존 엑셀 데이터부터 시작, 완벽한 인프라를 기다리지 않고 "있는 데이터로 즉시 시작" |
| **챔피언 번아웃** | 중간 | 중간 | 챔피언 활동을 정식 업무로 인정, 평가에 반영, 적절한 보상 |
| **AI 환각 리스크** | 중간 | 중간 | 모든 AI 산출물은 사람이 검증, 특히 숫자/가격/수량은 이중 확인 필수 |
| **브랜드 간 AX 격차** | 낮음 | 높음 | 순차 전개(와키윌리→커버낫→리), 선행 브랜드의 러닝을 후발에 전이 |

|--------|----------|

| **인력 이탈** | 인재 밀도 향상과 보상 연계 — "AI로 효율화 → 더 좋은 보상" 구조 |

## 10. 장기 비전

### 2027-2028 방향성

#### Horizon 1: AI 비주얼 생성 (2027 하반기~)

- **AI 촬영**: 실제 모델 + AI 배경/의상 교체로 촬영 비용 70% 절감
- **가상 피팅**: 자사몰에 AI 가상 피팅 기능 도입 (반품률 감소)
- **룩북 자동화**: BTA별 착장 조합 AI 자동 생성 → 디자이너는 큐레이션만

> **비케이브 규모 고려**: 글로벌 브랜드처럼 자체 AI 모델을 개발하기보다, 스튜디오랩 '젠시' 등 국내 AI 촬영 솔루션을 활용하여 투자 대비 효과를 극대화.

#### Horizon 2: 수요 예측 고도화 (2027~)

- **Nextail/Nul 수준의 SKU별 수요 예측**: 베이직 아이템부터 시작
- **자동 리오더**: 재고 감소 패턴 → AI가 리오더 추천 → MD 승인 → 자동 발주
- **날씨/이벤트 연동**: 기온·습도·연휴 등 외부 변수 반영 수요 조정

> **비케이브 규모 고려**: Heuritech이나 Nextail 같은 글로벌 플랫폼보다, 자사 판매 데이터 + 간단한 ML 모델로 시작. 데이터가 축적되면 단계적으로 고도화.

#### Horizon 3: AI 네이티브 조직 (2028~)

- **역할 재정의**: 중간 관리자 → "AI 오케스트레이터" (AI 에이전트를 조율하는 역할)
- **데이터 기반 평가**: AI가 업무 기여도를 객관적으로 분석, 컨티뉴어스 피드백
- **크로스 브랜드 인사이트**: 10개 브랜드 데이터를 AI가 통합 분석, 시너지 기회 발굴

> **비케이브 규모 고려**: 조직 플래트닝은 비케이브의 강소기업 특성상 자연스러운 방향. 대기업보다 빠르게 실현 가능.
