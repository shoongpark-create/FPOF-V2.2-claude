# 유저 페르소나 설계

> 와키윌리 UNI/WOMAN 페르소나를 리서치 데이터 기반으로 정교화하고, JTBD/Pains/Gains를 개발합니다 — pm-skills 원본 프레임워크를 FPOF 패션 하우스 컨텍스트에 맞게 커스터마이징

## 언제 사용
- 시즌 기획 시 타겟 고객 이해를 심화할 때
- 기존 UNI/WOMAN 페르소나를 데이터 기반으로 업데이트할 때
- 새로운 카테고리(용품, 글로벌) 진입 시 신규 페르소나를 개발할 때
- PDCA Plan 단계에서 타겟 정의를 정교화할 때

## FPOF 컨텍스트
- 이 스킬은 와키윌리 브랜드의 UNI/WOMAN 페르소나 정교화 및 신규 페르소나 개발에 활용합니다
- 관련 프리셋: `personas.json`(기존 UNI/WOMAN 페르소나, 고객 인사이트), `brand.config.json`(DNA, 비전, 타겟 정의)
- personas.json의 기존 페르소나를 출발점으로 활용하되, 리서치 데이터로 검증 및 확장합니다
- 코어 타겟(18~25세 자유로운 트렌드리더)의 JTBD와 숨겨진 인사이트를 발굴합니다

## 사전 준비
1. `.fpof-state.json` → 현재 시즌 확인
2. `personas.json` → 기존 UNI/WOMAN 페르소나 참조 (출발점)
3. `brand.config.json` → 코어 타겟 정의, 브랜드 DNA 참조
4. 리서치 데이터(서베이, 인터뷰, CSV, 구매 데이터 등)가 있다면 준비

## 실행 절차

### Step 1: 기존 페르소나 리뷰
personas.json의 현재 페르소나를 분석합니다.
- UNI 페르소나: 특성, 니즈, 행동 패턴 리뷰
- WOMAN 페르소나: 특성, 니즈, 행동 패턴 리뷰
- 기존 페르소나의 강점과 데이터 갭 식별

### Step 2: 데이터 수집 및 패턴 인식
제공된 리서치 데이터에서 패턴을 추출합니다.
- CSV, Excel, 서베이 응답, 인터뷰 트랜스크립트 분석
- 반복되는 특성, 목표, 페인 포인트, 행동 식별
- 인구통계뿐 아니라 행동 패턴에 주목
- 기존 페르소나와의 일치/불일치 확인

### Step 3: 페르소나 세분화 및 구축
유사한 사용자를 그룹화하여 정교한 페르소나를 구축합니다.

각 페르소나별 (최소 3개):

**페르소나 이름 & 인구통계**
- 연령대, 직업/역할, 라이프스타일, 핵심 특성
- 와키윌리 관계 (얼리어답터/충성고객/신규유입 등)

**Primary Job-to-be-Done**
- 이 페르소나가 달성하려는 핵심 과업
- 과업의 맥락과 빈도
- 성공의 기준

**Top 3 Pain Points**
- 과업 완수를 방해하는 구체적 장벽
- 각 페인의 심각도와 빈도

**Top 3 Desired Gains**
- 원하는 이득, 결과, 솔루션
- 성공 측정 방법

**1가지 예상 외 인사이트**
- 데이터에서 발견한 반직관적 행동 또는 동기
- 이것이 상품 기획에 갖는 의미

**와키윌리 상품 적합성**
- 이 페르소나의 니즈를 어떻게 충족하는가
- 잠재적 마찰점 또는 미충족 니즈

### Step 4: personas.json 업데이트 제안
기존 프리셋을 어떻게 갱신할지 제안합니다.
- UNI 페르소나 정교화 포인트
- WOMAN 페르소나 정교화 포인트
- 신규 페르소나 추가 필요 여부
- 글로벌 타겟 페르소나 확장

### Step 5: 와키윌리 전략 연결
페르소나 인사이트를 브랜드 전략에 연결합니다.
- 브랜드 DNA(Kitsch Street & IP Universe)와의 공명 포인트
- 5대 경영목표별 페르소나 시사점
- 시즌 기획(라인시트, 무드보드)에 반영할 인사이트
- 마케팅 채널/메시지 최적화 제안
