# 회의록 작성

> 바이어 미팅, 디자인 리뷰, 기획 회의 등의 트랜스크립트를 구조화된 회의록으로 정리 — pm-skills 원본 프레임워크를 FPOF 패션 하우스 컨텍스트에 맞게 커스터마이징

## 언제 사용
- 바이어 미팅 후 회의 내용 정리 시
- 디자인 리뷰 회의 요약 시
- 시즌 기획 회의, MD 회의 등의 회의록 작성 시
- 녹음/트랜스크립트 기반 구조화된 요약이 필요할 때

## FPOF 컨텍스트
- 와키윌리 시즌 사이클의 모든 PDCA 단계에서 사용 가능
- 패션 업계 특유의 회의 유형(바이어 프레젠테이션, 원단 미팅, 피팅 리뷰, 촬영 브리핑 등) 반영
- 액션 아이템은 시즌 일정과 연결하여 기한 설정

## 사전 준비
- `.fpof-state.json`에서 현재 시즌/단계 확인
- 사용자가 제공하는 회의 트랜스크립트, 녹음, 메모 파일 읽기

## 실행 절차

### Step 1: 회의 내용 수집
사용자가 회의 트랜스크립트, 녹음, 메모 파일을 제공하면 철저히 읽는다. 배경 자료가 필요하면 웹 검색으로 관련 정보를 파악한다.

### Step 2: 단계적 사고
- 누가 참석했고 각각의 역할은?
- 주요 의제/주제는?
- 어떤 결정이 내려졌는가?
- 다음 단계와 담당자는?
- 미해결 질문이나 블로커는?

### Step 3: 핵심 정보 추출
- 주요 논의 주제 식별
- 회의 중 내려진 결정 사항 기록
- 이견이나 우려 사항 표시
- 담당자와 기한이 있는 액션 아이템 도출

### Step 4: 구조화된 요약 작성

### Step 5: 명확한 언어 사용
누구나 이해할 수 있는 쉬운 용어를 사용한다. 전문 용어는 간략히 설명을 덧붙인다.

### Step 6: 와키윌리 적용 필터링
- 액션 아이템의 기한을 시즌 마일스톤과 연결
- 결정 사항이 시즌 산출물에 미치는 영향 표시
- 관련 에이전시/담당자를 FPOF 팀 구조에 맞게 매핑

## 완료 조건
- 모든 핵심 결정 사항이 빠짐없이 기록됨
- 액션 아이템에 담당자와 기한이 명시됨
- 미해결 사항이 별도로 정리됨
- 객관적인 톤으로 작성됨 (개인 의견 배제)
