# 테스트 시나리오 작성

> 자사몰/앱 기능 테스트, 주문 플로우 QA 등 유저 스토리 기반의 포괄적 테스트 시나리오 작성 — pm-skills 원본 프레임워크를 FPOF 패션 하우스 컨텍스트에 맞게 커스터마이징

## 언제 사용
- 자사몰/앱 신규 기능 검증용 테스트 시나리오 작성 시
- 주문/결제/배송 플로우 QA 계획 수립 시
- 유저 스토리의 수용 기준을 테스트 케이스로 변환할 때
- 시즌 런칭 전 기능 검증이 필요할 때

## FPOF 컨텍스트
- 와키윌리 자사몰과 앱의 핵심 기능(상품 검색, 장바구니, 주문, IP 콜라보 페이지 등) 테스트
- 시즌 런칭 전 Do 단계에서 QC 본부와 협업하여 실행
- 채널별(자사몰, 무신사, 29CM 등) 테스트 범위 설정

## 사전 준비
- `.fpof-state.json`에서 현재 시즌/단계 확인
- 테스트 대상 유저 스토리(제목, 수용 기준) 확인
- 사용자가 제공하는 추가 테스트 컨텍스트 또는 제약 조건 읽기

## 실행 절차

### Step 1: 유저 스토리 및 수용 기준 검토
유저 스토리와 수용 기준을 꼼꼼히 읽는다.

### Step 2: 테스트 목표 정의
어떤 구체적인 행동을 검증할지 정의한다.

### Step 3: 시작 조건 설정
시스템 상태, 데이터 세팅, 설정을 명시한다.

### Step 4: 사용자 역할 식별
테스트 액션을 수행하는 사용자가 누구인지 식별한다.

### Step 5: 테스트 단계 작성
상호작용을 단계별로 분해한다.

### Step 6: 기대 결과 정의
각 단계 후 관찰 가능한 결과를 정의한다.

### Step 7: 엣지 케이스 고려
잘못된 입력, 경계 조건, 예외 상황을 포함한다.

### Step 8: 와키윌리 적용 필터링
- 와키윌리 자사몰/앱의 핵심 플로우(IP 캐릭터 필터, 콜라보 랜딩, 사이즈 추천 등) 반영
- 모바일/데스크톱 양쪽 테스트 포함
- 시즌 런칭 전 필수 검증 항목과 우선순위 매핑

## 시나리오 템플릿
```markdown
**테스트 시나리오:** [명확한 시나리오명]

**테스트 목표:** [이 테스트가 검증하는 것]

**시작 조건:**
- [필요한 시스템 상태]
- [필요한 데이터 또는 설정]
- [사용자 세팅 또는 권한]

**사용자 역할:** [테스트를 수행하는 사용자]

**테스트 단계:**
1. [첫 번째 액션 및 기대 결과]
2. [두 번째 액션 및 관찰 가능한 결과]
3. [세 번째 액션 및 시스템 동작]
4. [완료 액션 및 최종 상태]

**기대 결과:**
- [관찰 가능한 결과 1]
- [관찰 가능한 결과 2]
- [관찰 가능한 결과 3]
```

## 예시 (와키윌리 컨텍스트)
```markdown
**테스트 시나리오:** 캐릭터별 상품 필터링 기능 검증

**테스트 목표:** 캐릭터 필터 선택 시 해당 콜라보 상품만 정확히 표시되는지 확인

**시작 조건:**
- 사용자가 자사몰에 로그인한 상태
- 카탈로그에 키키, 몽몽, 뿌뿌 캐릭터 콜라보 상품이 각각 5개 이상 등록됨
- 상품 리스트 페이지에 접속한 상태

**사용자 역할:** 온라인 쇼퍼 (UNI 페르소나)

**테스트 단계:**
1. 상품 리스트 페이지에서 캐릭터 필터 영역 확인 → 필터 옵션이 표시됨
2. '키키' 캐릭터 필터 선택 → 키키 콜라보 상품만 표시됨
3. '몽몽' 캐릭터 추가 선택 (복수 선택) → 키키 + 몽몽 상품 모두 표시됨
4. 모든 필터 해제 → 전체 상품으로 복귀
5. 상품이 없는 캐릭터 선택 → "해당 상품이 없습니다" 안내 표시

**기대 결과:**
- 캐릭터 필터가 상품 리스트 상단에 가시적으로 노출
- 필터 적용 시 0.5초 이내 결과 갱신
- 복수 캐릭터 선택 시 OR 조건으로 작동
- 필터 해제 시 전체 상품으로 즉시 복귀
- 결과 0건 시 사용자 친화적 안내 메시지 표시
- 모바일/데스크톱 모두 동일하게 작동
```

## 완료 조건
- 각 수용 기준에 대한 포괄적 테스트 시나리오 제시
- 테스트 목표가 유저 스토리 의도와 정합
- 단계별 상세 테스트 액션 포함
- 각 단계 후 관찰 가능한 기대 결과 명시
