# 스프린트 회고

> 시즌 종료 후 구조화된 회고를 진행하여 인사이트를 도출하고 개선 액션을 수립 — pm-skills 원본 프레임워크를 FPOF 패션 하우스 컨텍스트에 맞게 커스터마이징

## 언제 사용
- 시즌 종료 후 PDCA Check 단계에서 팀 회고 진행 시
- 스프린트/마일스톤 완료 후 팀 피드백 정리 시
- 이전 시즌 대비 개선점/유지점 도출 시
- Start/Stop/Continue, 4Ls, Sailboat 등 회고 포맷 활용 시

## FPOF 컨텍스트
- 와키윌리 시즌 사이클(PDCA) 종료 후 Check 단계에서 실행
- `brand.config.json`의 5대 경영목표 달성 여부를 회고 기준으로 활용
- 시즌 산출물(트렌드 브리프, 라인시트, IMC 전략 등) 완성도를 성과 지표로 참조

## 사전 준비
- `.fpof-state.json`에서 현재 시즌/단계 확인
- `brand.config.json`에서 5대 경영목표 및 KPI 확인
- 시즌 산출물 폴더(`output/[시즌]/`)의 완성 현황 파악
- 사용자가 제공하는 피드백, 매출 데이터, 팀 의견 파일 읽기

## 실행 절차

### Step 1: 컨텍스트 수집
사용자가 스프린트 데이터, 팀 피드백, 이전 회고 노트를 파일로 제공하면 철저히 읽는다. 시즌 산출물 완성 현황과 매출 성과를 함께 파악한다.

### Step 2: 회고 포맷 선택
컨텍스트에 맞는 포맷을 선택하거나 사용자가 고르게 한다:

**포맷 A — Start / Stop / Continue**:
- **Start**: 앞으로 시작해야 할 것은?
- **Stop**: 중단해야 할 것은?
- **Continue**: 잘 되고 있어서 계속해야 할 것은?

**포맷 B — 4Ls (Liked / Learned / Lacked / Longed For)**:
- **Liked**: 팀이 좋았던 점은?
- **Learned**: 새로 배운 점은?
- **Lacked**: 부족했던 점은?
- **Longed For**: 있었으면 했던 점은?

**포맷 C — Sailboat**:
- **Wind (추진력)**: 우리를 앞으로 나아가게 한 것은?
- **Anchor (저항력)**: 우리를 느리게 한 것은?
- **Rocks (리스크)**: 앞에 놓인 위험은?
- **Island (목표)**: 우리가 도달하려는 곳은?

### Step 3: 피드백 정리
사용자가 원시 피드백(포스트잇, 설문 응답, 슬랙 메시지 등)을 제공하면:
- 유사 항목을 테마별로 그룹핑
- 가장 자주 언급된 주제 식별
- 감정 패턴 파악 (좌절, 에너지, 혼란)

### Step 4: 시즌 성과 분석
- 시즌 목표: 달성 / 부분 달성 / 미달성?
- 5대 경영목표 대비 실적 (매출 비중, 인지도, ROAS 등)
- 주요 블로커와 해결 방법
- 팀 협업 패턴 (무엇이 효과적이었고, 무엇이 아니었는지)

### Step 5: 우선순위 액션 아이템 생성

| 우선순위 | 액션 아이템 | 담당자 | 기한 | 성공 지표 |
|---|---|---|---|---|
| 1 | [구체적이고 실행 가능한 개선 사항] | [이름/역할] | [날짜] | [효과 확인 방법] |

- 2~3개로 제한 (더 많으면 실행되지 않음)
- 각각 구체적이고, 담당자 지정 가능하며, 측정 가능해야 함
- 이전 회고 액션이 있다면 완료 여부 참조

### Step 6: 와키윌리 적용 필터링
- 액션 아이템이 5대 경영목표 중 어떤 것과 연결되는지 매핑
- 다음 시즌 PDCA Plan 단계에 반영할 개선점 명시
- 패션 업계 특성(시즌 주기, 리드타임, 샘플 일정 등) 반영 여부 점검

## 산출물 포맷
```markdown
# [시즌] 회고 — [날짜]

## 시즌 성과
- 목표: [달성 / 부분 달성 / 미달성]
- 5대 경영목표 대비: [요약]

## 회고 포맷: [Start/Stop/Continue | 4Ls | Sailboat]

### [포맷별 섹션]
...

## 핵심 테마
1. [테마] — [요약]

## 액션 아이템
| 우선순위 | 액션 | 담당자 | 기한 | 성공 지표 |
|---|---|---|---|---|
| 1 | ... | ... | ... | ... |

## 이전 회고 캐리오버
- [이전 액션] — [상태: 완료 / 진행 중 / 미시작]

## 경영목표 연결
- [액션 아이템 ↔ 경영목표 매핑]
```

## 완료 조건
- 선택한 회고 포맷에 맞는 구조화된 결과물 제시
- 우선순위가 매겨진 2~3개의 구체적 액션 아이템 포함
- 각 액션에 담당자, 기한, 성공 지표 명시
- 이전 회고 액션의 이행 여부 확인 (있는 경우)

## 체크리스트
- [ ] 시즌 성과가 5대 경영목표 기준으로 평가되었는가?
- [ ] 액션 아이템이 2~3개로 제한되었는가?
- [ ] 각 액션이 구체적이고, 담당자 지정 가능하며, 측정 가능한가?
- [ ] 이전 회고 액션의 완료 여부를 확인했는가?
- [ ] 다음 시즌 PDCA 반영 사항이 명시되었는가?
- [ ] 건설적인 톤을 유지했는가? (비난이 아닌 개선 목적)
