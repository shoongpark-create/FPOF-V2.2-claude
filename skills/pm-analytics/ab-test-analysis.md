# A/B 테스트 결과 분석

> A/B 테스트 결과를 통계적으로 분석하고 Ship/Extend/Stop 의사결정을 내립니다 — pm-skills 원본 프레임워크를 FPOF 패션 하우스 컨텍스트에 맞게 커스터마이징

## 언제 사용
- 자사몰/무신사 상품 페이지 A/B 테스트 결과를 해석할 때
- 카피, 비주얼, CTA 등 마케팅 요소의 실험 결과를 분석할 때
- 전환율/클릭률/구매율 등 핵심 지표의 통계적 유의성을 검증할 때
- PDCA Check 단계에서 캠페인/페이지 최적화 근거를 마련할 때

## FPOF 컨텍스트
- 이 스킬은 와키윌리 브랜드의 자사몰, 무신사 등 온라인 채널에서 진행한 A/B 테스트 분석에 활용합니다
- 관련 프리셋: `channels.json`(6개 유통채널 매출/목표/성장률)
- 패션 이커머스 특성상 상품 상세페이지(PDP), 카피, 썸네일, 가격 표기 방식 등이 주요 테스트 대상입니다
- 코어 타겟(18~25세)의 모바일 쇼핑 행태를 고려한 해석이 필요합니다

## 사전 준비
1. `.fpof-state.json` → 현재 시즌 확인
2. `channels.json` → 테스트 대상 채널의 매출 구조 및 트래픽 참조
3. 테스트 데이터(CSV, Excel, GA 리포트 등) 준비
4. 실험 가설, 테스트 기간, 트래픽 분배 비율 확인

## 실행 절차

### Step 1: 실험 이해
테스트의 기본 설계를 파악합니다.
- 가설은 무엇인가? (예: "키키 캐릭터 썸네일이 일반 모델 컷보다 CTR이 높을 것이다")
- 변형(Variant)은 무엇인가?
- 1차 지표(Primary Metric)는 무엇인가? 가드레일 지표는?
- 테스트 기간은 얼마나 되었는가?
- 트래픽 분배 비율은?

### Step 2: 테스트 설계 검증
테스트의 통계적 유효성을 점검합니다.
- **표본 크기**: 기대 효과 크기 대비 충분한 표본인가?
  - 공식: n = (Z²α/2 x 2 x p x (1-p)) / MDE²
  - 통계적 검정력(Power) < 80%면 경고 표시
- **기간**: 최소 1~2 비즈니스 사이클(패션의 경우 주말 포함 필수) 경과했는가?
- **무작위 배정**: 샘플 비율 불일치(SRM) 징후는 없는가?
- **신규성/초두 효과**: 초기 행동 변화가 안정화될 충분한 시간이 있었는가?

### Step 3: 통계적 유의성 계산
핵심 통계 지표를 산출합니다.
- **전환율**: 컨트롤 vs 변형
- **상대적 리프트**: (변형 - 컨트롤) / 컨트롤 x 100
- **p-value**: 양측 z-test 또는 카이제곱 검정
- **신뢰구간**: 차이에 대한 95% CI
- **통계적 유의성**: p < 0.05인가?
- **실무적 유의성**: 리프트가 비즈니스에 의미 있는 수준인가?

원시 데이터가 제공되면 Python 스크립트를 생성하여 계산합니다.

### Step 4: 가드레일 지표 확인
핵심 보호 지표의 악화 여부를 점검합니다.
- 매출, 객단가, 이탈률, 페이지 로딩 시간 등 가드레일 지표에 부정적 영향은 없는가?
- 1차 지표가 개선되었더라도 가드레일 악화 시 진정한 승리가 아닐 수 있음

### Step 5: 결과 해석 및 의사결정

| 결과 | 권고 |
|------|------|
| 유의미한 양의 리프트, 가드레일 이상 없음 | **Ship** — 100% 롤아웃 |
| 유의미한 양의 리프트, 가드레일 우려 | **Investigate** — 트레이드오프 분석 후 결정 |
| 유의하지 않음, 양의 추세 | **Extend** — 더 많은 데이터 필요 |
| 유의하지 않음, 플랫 | **Stop** — 의미 있는 차이 없음 |
| 유의미한 음의 리프트 | **Don't Ship** — 컨트롤 유지, 원인 분석 |

### Step 6: 와키윌리 적용 필터링
분석 결과를 와키윌리 브랜드 렌즈로 해석합니다.
- 코어 타겟(18~25세) 세그먼트에서의 결과는 별도 확인
- IP 캐릭터(키키 등) 활용 변형의 브랜드 정합성 평가
- 채널별 차이 (자사몰 vs 무신사 vs 29CM 등) 고려
- 시즌성 요인 반영 (패션 성수기/비수기 트래픽 차이)
