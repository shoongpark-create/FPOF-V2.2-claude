# 문법 및 흐름 체크

> 산출물의 문법, 로직, 흐름을 점검하고 브랜드 톤 정합성을 확인합니다 — pm-skills 원본 프레임워크를 FPOF 패션 하우스 컨텍스트에 맞게 커스터마이징

## 언제 사용
- FPOF 산출물(카피, 보고서, 기획서 등)의 문법/맞춤법을 검수할 때
- 논리적 흐름과 구조의 완성도를 점검할 때
- 브랜드 톤앤매너 정합성을 확인할 때
- 모든 PDCA 단계에서 산출물 품질을 높일 때

## FPOF 컨텍스트
- 이 스킬은 와키윌리 브랜드의 모든 산출물 품질 검수에 활용합니다
- 관련 프리셋: `tone-manner.json`(톤앤매너, 채널별 톤, 브랜드 어휘)
- 고객 대면 콘텐츠(PDP 카피, SNS 캡션, 캠페인 카피)는 반드시 tone-manner.json 기준으로 검수
- 내부 문서(기획서, 보고서)는 명확성과 논리 흐름 중심으로 검수

## 사전 준비
1. `.fpof-state.json` → 현재 시즌/단계 확인
2. `tone-manner.json` → 브랜드 톤앤매너, 채널별 톤, 브랜드 어휘/금지 어휘 참조
3. 검수 대상 텍스트 및 목적(마케팅 카피/기획서/보고서 등) 확인
4. 대상 독자(고객/내부/파트너) 확인

## 실행 절차

### Step 1: 컨텍스트 파악
검수 대상의 목적과 독자를 이해합니다.
- 문서 유형: 마케팅 카피, 기획서, 보고서, 이메일, SNS 콘텐츠
- 대상 독자: 코어 타겟(18~25세), 내부 팀, 파트너/공급업체
- 기대 톤: `tone-manner.json` 기준 채널별 톤 확인
- 문서의 목적과 핵심 메시지

### Step 2: 오류 스캔
텍스트를 통독하며 오류를 식별합니다.
- **문법 오류**: 맞춤법, 띄어쓰기, 조사 사용, 시제 일관성, 수식어 위치
- **논리 오류**: 모순, 근거 없는 주장, 불명확한 인과관계, 불완전한 논지
- **흐름 오류**: 어색한 전환, 불명확한 구조, 중복 표현, 모호한 지시어

### Step 3: 오류 분류
발견된 오류를 유형별로 정리합니다.
1. 문법 (맞춤법, 띄어쓰기, 문장 구조)
2. 논리 (명확성, 일관성, 논거)
3. 흐름 (전환, 문장 구조, 가독성, 톤 일관성)

### Step 4: 수정 제안 작성
각 오류에 대해 구체적인 수정안을 제시합니다.
- **위치**: 텍스트 내 위치 (예: "3단락, 2번째 문장")
- **오류 내용**: 무엇이 잘못되었는지
- **수정 제안**: 어떻게 고칠 것인지
- **근거**: 왜 중요한지 (명확성, 문법 규칙, 흐름, 톤)

### Step 5: 우선순위 결정
가장 영향력 있는 이슈를 먼저 표시합니다.
- **Critical**: 독자를 혼란스럽게 하는 문법/논리 오류
- **Important**: 가독성을 해치는 흐름 이슈
- **Minor**: 스타일 제안 또는 다듬기

### Step 6: 브랜드 톤 정합성 검수
와키윌리 브랜드 톤과의 일치 여부를 확인합니다.
- `tone-manner.json`의 브랜드 어휘 사용 여부
- 금지 어휘 사용 여부
- 채널별 적합한 톤 사용 여부 (SNS vs 자사몰 vs PR)
- Kitsch/Street 감성과의 부합성
- 코어 타겟(18~25세) 언어 감각과의 일치
