# 인사이트 도출 & 아카이빙

> 인사이트 아키텍트가 성공/실패 사례를 분석하고 재생산 가능한 지식으로 축적하는 전문 매뉴얼

## 언제 사용
- "왜 잘 팔렸어?", "실패 원인 분석해줘", "플레이북 만들어줘"
- Check 단계에서 매출 분석 완료 후

## 사전 준비
1. `output/[시즌]/check/sales-analysis.md` → 매출 분석 결과 (필수 선행)
2. `knowledge/` → 기존 축적된 인사이트 (이전 시즌)
3. `presets/wacky-willy/brand.config.json` → 경영목표와 연결

## 성공 사례 분석 프레임워크
```
1. 무엇이 성공했는가? (What) → 구체적 수치
2. 왜 성공했는가? (Why)
   → 외부 요인: 트렌드 타이밍, 시장, 경쟁사
   → 내부 요인: 상품력, 가격, 마케팅, 타이밍
   → 우연 vs 전략: 의도한 결과인가?
3. 반복 가능한가? (Reproducible?) → 재현 조건
4. 원리화 → "~한 조건에서 ~하면 ~한 결과를 기대할 수 있다"
```

## 실패 사례 분석 프레임워크
```
1. 무엇이 실패했는가? (What) → 목표 대비 미달 수치
2. 왜 실패했는가? (Why)
   → 예측 실패 / 실행 실패 / 구조 실패
3. 예방 가능한가? (Preventable?) → 시그널/의사결정
4. 교훈 → "~한 상황에서는 ~하지 말아야 한다"
```

## 감성 분석 프레임워크 (PM-Skills 통합)

대규모 고객 피드백(무신사 리뷰, SNS 댓글, NPS 서베이, CS 문의)을 체계적으로 분석합니다.

### 감성 분석 실행 절차

1. **데이터 수집**: 피드백 소스별 데이터 수집 (무신사 리뷰, 인스타 댓글, CS 문의, NPS)
2. **세그먼트 분류**: 고객 세그먼트별로 피드백 그룹핑
   - personas.json의 UNI/WOMAN 페르소나 기준
   - 채널별 (자사몰/무신사/오프라인)
   - 상품 카테고리별
3. **감성 스코어링**: 각 세그먼트의 감성 점수 산출
   - 스코어 범위: -1.0(매우 부정) ~ +1.0(매우 긍정)
   - 기능적 만족도 vs 감성적 만족도 분리
4. **테마 추출**: 반복되는 키워드/패턴을 테마로 클러스터링
   - 만족 드라이버 (Satisfaction Drivers): 재구매/추천 이유
   - 페인 포인트 (Pain Points): 불만/이탈 사유
   - 미충족 니즈 (Unmet Needs): 원하지만 없는 것
5. **인사이트 도출**: 테마 → 액셔너블 인사이트 변환

### 감성 분석 산출물

```markdown
## 감성 분석 요약

### 세그먼트별 감성 스코어
| 세그먼트 | 감성 점수 | 샘플 수 | 만족 드라이버 | 페인 포인트 |
|----------|----------|---------|-------------|------------|

### Top 테마 (빈도순)
| 순위 | 테마 | 빈도 | 감성 | 카테고리 | 액션 |
|------|------|------|------|---------|------|

### 인사이트 카드 연결

```
