# Executive Summary Skill (업무용 요약 보고서)

회의록, 이메일, 마케팅 리포트, 대화 로그 등 긴 텍스트를 **회사 양식에 맞춘 구조화된 요약 보고서**로 변환하는 스킬입니다.

## 트리거 조건

다음 키워드/문맥이 포함된 요청에서 이 스킬을 **우선 적용**합니다:

| 키워드 | 사용 예시 |
|--------|----------|
| `주간 보고` | "이 회의록 주간 보고용으로 정리해 줘" |
| `임원 보고` | "임원 보고용 요약으로 만들어 줘" |
| `캠페인 결과 요약` | "이번 캠페인 결과 요약해 줘" |
| `프로젝트 회의 요약` | "프로젝트 회의 요약 부탁해" |
| `보고용으로 요약` | "이 내용 보고용으로 요약해 줘" |
| `보고서 작성` | "마케팅 보고서 작성해 줘" |

> [!tip] 자세한 트리거 키워드 목록
> `references/trigger-keywords.md` 참조

## 작업 절차 (4단계)

### 1단계: 정보 추출 (Extraction)

원본 텍스트에서 다음 항목을 **식별하고 추출**합니다:

```
□ 핵심 목표 및 배경
□ 정량적 수치/지표 (CTR, 전환율, 매출, ROI 등)
□ 주요 결정사항 및 합의 내용
□ 발견된 이슈/리스크
□ 할 일(Action Items) 및 담당자, 마감일
□ 미결 사항 또는 추가 논의 필요 항목
```

> [!note] 지표 정의 참고
> 마케팅/프로젝트 관련 주요 지표 정의는 `references/metrics-definition.md` 참조

---

### 2단계: 구조화된 재정리 (Structuring)

추출한 정보를 **아래 고정 섹션 구조**로 재정리합니다:

#### 📋 출력 섹션 구조

```markdown
## Summary
[3줄 이내 핵심 요약 - 목적, 결과, 다음 단계]

## Key Metrics
| 지표 | 값 | 전기 대비 | 비고 |
|------|-----|----------|------|
| ... | ... | ... | ... |

## Insights
- [결과의 원인 분석]
- [시사점 및 의미 해석]

## Action Items
- [ ] [할 일] | 담당: [이름] | 마감: [YYYY-MM-DD]
- [ ] ...

## 추가로 필요한 정보 (선택)
- [누락된 정보에 대한 질문]
```

> [!tip] 템플릿 참조
> 전체 템플릿은 `templates/report-template.md` 참조

---

### 3단계: 문체 및 표기 규칙 (Styling)

| 규칙 | 예시 |
|------|------|
| **경어체** 사용 | "~입니다", "~해 주세요" |
| **한영 병기** | "전환율(Conversion Rate)" |
| **숫자 표기** | 천 단위 쉼표: "1,234,567원" |
| **날짜 형식** | `YYYY-MM-DD` (예: 2026-01-15) |
| **백분율** | 소수점 1자리: "12.5%" |

---

### 4단계: 누락 정보 확인 (Validation)

원본에서 다음 정보가 **누락된 경우**, `## 추가로 필요한 정보` 섹션에 질문 형태로 정리합니다:

- 담당자가 명시되지 않은 Action Item
- 마감일이 없는 할 일
- 맥락 없이 언급된 수치
- 비교 기준이 없는 성과 지표

```markdown
## 추가로 필요한 정보
1. "신규 캠페인 A/B 테스트"의 담당자는 누구인가요?
2. CTR 3.2%의 비교 기준(전기/목표치)은 무엇인가요?
3. "Q1 마감" 언급이 있는데, 정확한 날짜는 언제인가요?
```

---

## 출력 형식 규칙

### Markdown 필수 사용

- 모든 출력은 **Markdown 형식**
- 표는 `|` 파이프 구분자 사용
- Action Items는 **체크박스 리스트** 필수: `- [ ]`

### Action Item 형식

```markdown
- [ ] [구체적인 할 일] | 담당: [이름] | 마감: [YYYY-MM-DD]
```

예시:
```markdown
- [ ] 랜딩페이지 A/B 테스트 결과 분석 | 담당: 김마케터 | 마감: 2026-01-20
- [ ] 광고 소재 3종 제작 요청 | 담당: 디자인팀 | 마감: 2026-01-22
```

## 품질 기준

### ✅ 좋은 보고서 (GOOD)

- 수치가 표로 명확히 정리됨
- Action Item에 담당자와 마감일이 **모두** 있음
- Insights에 "왜"에 대한 분석이 포함됨
- 원문 대비 70% 이상 압축됨

### ❌ 나쁜 보고서 (BAD)

- 원문을 거의 그대로 복붙
- 수치를 장문으로만 설명 (표 없음)
- Action Item에 담당자/마감일 누락
- "좋았다", "잘됐다" 등 모호한 표현만 사용

> `templates/example-good.md` 참조

## 입력 유형별 처리 가이드

| 입력 유형 | 주요 추출 포인트 |
|-----------|-----------------|
| **회의록** | 결정사항, 논의 내용, 액션 아이템, 참석자 |
| **이메일 스레드** | 핵심 요청, 승인 사항, 마감일, 담당자 |
| **마케팅 리포트** | KPI 수치, 캠페인 성과, 개선점, 예산 |
| **Slack/Teams 로그** | 결론, 합의사항, 멘션된 담당자, 기한 |

## 자주 사용되는 지표 (Quick Reference)

| 약어 | 풀네임 | 설명 |

| LTV | Lifetime Value | 고객 생애 가치 |
