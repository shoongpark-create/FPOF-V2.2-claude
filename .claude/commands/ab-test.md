# A/B 테스트 분석 (A/B Test Analysis)

실험 결과를 통계적으로 분석하고 Ship/Extend/Stop 의사결정을 도출합니다.

## 사용법
`/ab-test <테스트 결과 데이터>` — 요약 통계, CSV, 스크린샷, 설명 등 모든 형식을 지원합니다.

## 절차

### Step 1: 테스트 데이터 수집
1. `.fpof-state.json`에서 현재 시즌과 단계 확인
2. 입력 형식:
   - 요약 통계 (전환율, 표본 크기)
   - 원시 데이터 CSV (user_id, variant, converted, timestamp)
   - 실험 플랫폼 스크린샷
   - 실험 설명 및 결과

### Step 2: 테스트 설계 검증
분석 전 확인:
- 표본 크기 충분성 (검정력 분석)
- 테스트 기간 적정성 (최소 1~2 비즈니스 사이클)
- 무작위화 품질 (표본 비율 불일치 체크)
- 테스트 기간 중 외부 요인 유무

### Step 3: 결과 분석
`skills/pm-analytics/ab-test-analysis.md` 참조:
- **통계적 유의성**: p-value, 신뢰구간 계산
- **효과 크기**: 절대적/상대적 차이
- **실질적 유의성**: 비즈니스에 의미 있는 크기인가?
- **세그먼트 분석**: 세그먼트별 차별적 효과 (데이터 허용 시)

### Step 4: 의사결정 도출
- **SHIP**: 통계적·실질적으로 유의미 → 전체 적용
- **EXTEND**: 유의하지 않지만 검정력 부족 → 테스트 연장
- **STOP**: 효과 없음 또는 부정적 → 중단
- 비즈니스 임팩트 추정 (매출, 전환율 변화)
- 주의사항 (신기효과, 세그먼트 차이, 외부 편향)

### Step 5: 산출물 저장
산출물을 `output/[시즌]/_season/check_ab-test-analysis.md`에 저장한다.
`.fpof-state.json`의 artifacts 배열에 파일 경로 추가.

### Step 6: 다음 단계 제안
- "이 결과를 바탕으로 **후속 실험을 설계**하시겠어요?"
- "특정 세그먼트에 대해 **심층 분석**을 하시겠어요?"
- "런칭 후 모니터링할 **SQL 쿼리**를 작성하시겠어요?"
