# 고객 피드백 분석 (Feedback Analysis)

고객 피드백 데이터에서 감성 분석, 테마 추출, 세그먼트별 인사이트를 도출합니다.

## 사용법
`/analyze-feedback <피드백 데이터>` — CSV, 텍스트, 파일 등 모든 형식을 지원합니다.

## 절차

### Step 1: 피드백 데이터 수집
1. `.fpof-state.json`에서 현재 시즌과 단계 확인
2. `presets/wacky-willy/personas.json` 참조 — 세그먼트 기준
3. 사용자에게 확인:
   - 피드백 유형 (리뷰 / CS 데이터 / 설문 / SNS 댓글 / NPS)
   - 세그먼트별 분석 필요 여부 (채널별 / 고객 등급별 / 지역별)
   - 분석 목적 (전체 트렌드 / 특정 이슈 / 시간에 따른 변화)

### Step 2: 감성 분석
`skills/data/insight-archiving.md` (감성 분석 내장) 참조:
- **감성 스코어링**: 각 피드백을 긍정/중립/부정으로 분류
- **테마 추출**: 반복되는 주제를 식별하고 관련 피드백을 클러스터링
- **빈도 분석**: 각 테마의 출현 빈도
- **세그먼트 분석**: 사용자 세그먼트별 감성과 테마 분석
- **트렌드 감지**: 날짜 데이터가 있는 경우 감성 변화 추이

### Step 3: 분석 보고서 작성
- 전체 감성 요약 (긍정/중립/부정 비율, 평균 감성 점수, 추이)
- Top 테마별 심층 분석 (언급 수, 감성, 영향받는 세그먼트, 근본 원인, 권고)
- 세그먼트별 분석표
- 주목할 만한 인용구
- 실행 가능한 인사이트 리스트

### Step 4: 산출물 저장
산출물을 `output/[시즌]/_season/check_feedback-analysis.md`에 저장한다.
`.fpof-state.json`의 artifacts 배열에 파일 경로 추가.

### Step 5: 다음 단계 제안
- "피드백 패턴으로 **사용자 페르소나**를 만드시겠어요?"
- "주요 테마를 **기능 요청으로 트리아지**하시겠어요?"
- "특정 테마를 심층 조사할 **인터뷰 스크립트**를 만드시겠어요?"
