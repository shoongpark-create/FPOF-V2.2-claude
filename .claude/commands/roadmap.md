# 아웃컴 로드맵 변환 (Outcome Roadmap)

기능 중심 로드맵을 아웃컴(성과) 중심 로드맵으로 변환합니다. "무엇(What)" 대신 "왜(Why)"를 전달합니다.

## 사용법
`/roadmap <기능 리스트 또는 기존 로드맵>` — 텍스트, 파일, 스프레드시트 등 모든 형식을 지원합니다.

## 절차

### Step 1: 현재 로드맵 수집
1. `.fpof-state.json`에서 현재 시즌과 단계 확인
2. `presets/wacky-willy/brand.config.json` 참조 — 전략 목표
3. 입력 수용:
   - 기능 리스트 / 백로그 아이템
   - 로드맵 문서 (Now/Next/Later, 분기별, 타임라인)
   - 스프레드시트 / 간트 차트
4. 각 항목에서 추출: 기능명, 설명, 목표 시기, 맥락

### Step 2: 전략적 맥락 파악
사용자에게 확인:
- 이 기간의 목표 또는 OKR
- 로드맵의 청중 (경영진 / 개발팀 / 고객 / 이사회)
- 선호 포맷 (Now/Next/Later / 분기별 / 타임라인)

### Step 3: 각 항목 변환
`skills/pm-execution/outcome-roadmap.md` 참조:

각 기능/아웃풋에 대해:
1. 달성하려는 **사용자 또는 비즈니스 아웃컴** 식별
2. 아웃컴 문장으로 재작성: "[동사] [세그먼트]의 [지표/경험]"
3. 동일 아웃컴을 위한 기능들을 하나의 이니셔티브로 그룹화
4. 각 아웃컴에 성공 지표 추가

**변환 예시**:
- "IP 콜라보 그래픽 티 개발" → "코어타겟의 브랜드 접점을 확대 — 목표: IP 상품 매출 비중 15%"
- "자사몰 UX 개편" → "온라인 구매 전환율 향상 — 목표: 자사몰 CVR 2.5%"

### Step 4: 변환된 로드맵 생성
Now/Next/Later 포맷으로 전략 테마별 아웃컴 정리.
변환 노트 (원래 기능 → 변환된 아웃컴 → 변환 이유) 포함.

### Step 5: 산출물 저장
산출물을 `output/[시즌]/_season/plan_outcome-roadmap.md`에 저장한다.
`.fpof-state.json`의 artifacts 배열에 파일 경로 추가.

### Step 6: 다음 단계 제안
- "각 아웃컴에 **OKR 정렬**을 추가하시겠어요?"
- "이 로드맵으로 **이해관계자 프레젠테이션**을 만드시겠어요?"
- "Now 항목의 **리스크를 식별**하시겠어요?"
