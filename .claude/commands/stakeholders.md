# 이해관계자 맵 (Stakeholder Map)

이해관계자를 Power x Interest 매트릭스에 매핑하고 맞춤 커뮤니케이션 계획을 수립합니다.

## 사용법
`/stakeholders <프로젝트 또는 이니셔티브>` — 대상을 생략하면 현재 시즌 전체 기준으로 매핑합니다.

## 절차

### Step 1: 이니셔티브 파악
1. `.fpof-state.json`에서 현재 시즌과 단계 확인
2. 사용자에게 확인:
   - 프로젝트 또는 이니셔티브 내용
   - 현재 단계 (기획 / 개발 / 런칭 / 런칭 후)
   - 이미 알고 있는 이해관계자
   - 정치적으로 민감한 요소 유무

### Step 2: 이해관계자 식별
`skills/pm-execution/stakeholder-map.md` 참조:
- **내부**: 디자인, 생산, 마케팅, 영업, 경영진, 재무, 법무
- **외부**: 고객, 파트너, 유통사, 인플루언서
- **놓치기 쉬운**: CS팀, 물류팀, 홍보팀, 문서팀

### Step 3: Power x Interest 그리드 매핑
- **Manage Closely** (고권력, 고관심): 긴밀 관리
- **Keep Satisfied** (고권력, 저관심): 만족 유지
- **Keep Informed** (저권력, 고관심): 정보 제공
- **Monitor** (저권력, 저관심): 모니터링

### Step 4: 커뮤니케이션 계획 수립
각 사분면별 커뮤니케이션 계획:
- 이해관계자, 채널, 빈도, 콘텐츠, 담당자
- 잠재적 이해충돌 및 완화 전략
- 에스컬레이션 경로
- RACI 매트릭스

### Step 5: 산출물 저장
산출물을 `output/[시즌]/_season/plan_stakeholder-map.md`에 저장한다.
`.fpof-state.json`의 artifacts 배열에 파일 경로 추가.

### Step 6: 다음 단계 제안
- "'Manage Closely' 그룹에 보낼 **첫 이해관계자 업데이트**를 작성하시겠어요?"
- "핵심 이해관계자 미팅용 **브리프**를 만드시겠어요?"
- "정기적인 **커뮤니케이션 체크리스트**를 설정하시겠어요?"
