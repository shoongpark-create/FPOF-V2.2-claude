# North Star Metric 정의 (North Star)

상품이 전달하는 핵심 가치를 포착하는 North Star Metric과 인풋 메트릭을 정의합니다.

## 사용법
`/north-star <대상 상품 또는 비즈니스>` — 대상을 생략하면 와키윌리 브랜드 기준으로 정의합니다.

## 절차

### Step 1: 상품 이해
1. `.fpof-state.json`에서 현재 시즌과 단계 확인
2. `presets/wacky-willy/brand.config.json` 참조 — 비전, 경영목표
3. `presets/wacky-willy/channels.json` 참조 — 매출 구조
4. 사용자에게 확인:
   - 대상 상품/비즈니스
   - 비즈니스 모델 (거래 / 구독 / 광고 / 마켓플레이스)
   - 현재 추적 중인 메트릭 (있는 경우)
   - 필요 계기 (신규 / 기존 메트릭 부적합 / 팀 정렬)

### Step 2: 비즈니스 게임 분류
`skills/data/sales-analysis.md` (NSM 내장) 참조:
- **어텐션**: 사용자 시간/참여에서 수익 (미디어, 소셜)
- **트랜잭션**: 구매에서 수익 (이커머스, 마켓플레이스)
- **프로덕티비티**: 효율 향상에서 수익 (SaaS, 도구)

### Step 3: North Star 정의
- 2~3개 North Star 후보 제안
- 7대 기준으로 검증:
  1. 고객에게 전달하는 가치를 표현하는가
  2. 매출의 선행 지표인가
  3. 측정 가능하고 추적 가능한가
  4. 전체 팀이 이해할 수 있는가
  5. 실행 가능한가 (팀이 영향을 줄 수 있는가)
  6. 허영 지표가 아닌가
  7. 실제 가치 전달 없이 게이밍 불가능한가
- 가장 강한 후보 권고 + 근거

### Step 4: 인풋 메트릭 정의
- North Star를 구동하는 3~5개 인풋 메트릭
- 각 메트릭이 특정 팀에서 실행 가능
- 인과 체인: 인풋 → North Star → 비즈니스 성과

### Step 5: 카운터 메트릭 및 안티패턴
- Goodhart의 법칙 방지를 위한 카운터 메트릭
- DAU, 매출 등 흔하지만 결함 있는 메트릭을 선택하지 않은 이유

### Step 6: 산출물 저장
산출물을 `output/[시즌]/_season/plan_north-star.md`에 저장한다.
`.fpof-state.json`의 artifacts 배열에 파일 경로 추가.
