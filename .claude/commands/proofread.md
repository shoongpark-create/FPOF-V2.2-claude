# 문법 및 흐름 체크 (Proofread)

텍스트의 문법, 논리, 흐름 오류를 식별하고 최소한의 타겟 수정안을 제시합니다.

## 사용법
`/proofread <검수할 텍스트>` — 텍스트 붙여넣기 또는 문서 파일을 지원합니다.

## 절차

### Step 1: 텍스트 수집
1. 입력 수용: 텍스트 붙여넣기, 문서 업로드 (DOCX, PDF, MD), 이메일 초안
2. 고객 대면 콘텐츠인 경우 `presets/wacky-willy/tone-manner.json` 참조하여 브랜드 보이스 기준도 확인

### Step 2: 3가지 카테고리 분석
`skills/pm-toolkit/grammar-check.md` 참조:

**문법(Grammar)**: 맞춤법, 띄어쓰기, 주술 호응, 시제 일관성, 조사 사용
**논리(Logic)**: 모순, 근거 없는 주장, 순환 논법, 불명확한 참조
**흐름(Flow)**: 어색한 전환, 문장 리듬, 단락 구조, 중복, 가독성

### Step 3: 이슈 보고
각 이슈별:
- **위치**: 이슈가 있는 텍스트 인용
- **이슈**: 무엇이 잘못되었는지
- **수정**: 수정된 텍스트

카테고리별 이슈 수 요약 및 전체 품질 평가.

### Step 4: 후속 제안
- "모든 수정을 **적용한 클린 텍스트**를 받으시겠어요?"
- "특정 섹션을 **더 상세히 검토**하시겠어요?"

## 원칙
- 최소한의 수정 — 필요한 것만 변경하고 저자의 목소리를 보존
- 스타일 취향과 오류를 구분 — 다른 것과 틀린 것은 다름
- 브랜드 콘텐츠일 경우 톤 일관성도 체크
