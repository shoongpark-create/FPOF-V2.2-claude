# 세션 14: 디스커버리 & 아이디에이션

> **소요시간**: 30분 (설명 10분 + 데모 10분 + 실습 10분)
> **대상**: 기획·전략·디자인
> **목표**: 아이디어→가설→실험 전체 디스커버리 사이클을 실습

---

## 1. 디스커버리 스킬 구조

```
/discover ── 전체 디스커버리 사이클 (원스탑)
  ├── /brainstorm ── 아이디어 발산
  ├── 가정 식별 ── 리스크 가정 도출
  ├── 가정 우선순위 ── Impact × Risk 매트릭스
  ├── 실험 설계 ── 검증 실험 설계
  └── /triage ── 기능 요청 정리
            ↓
/pre-mortem ── 런칭 전 리스크 사전 점검
```

---

## 2. `/discover` — 디스커버리 사이클 (원스탑)

### 사용법

```
/discover [주제]
```

5단계를 **순차 가이드**로 진행:

| 단계 | 내용 | 산출물 |
|------|------|--------|
| 1. 기회 탐색 | Opportunity Solution Tree로 고객 기회 구조화 | 기회 트리 |
| 2. 아이디어 발산 | 멀티 관점 브레인스토밍 | 아이디어 리스트 |
| 3. 가정 식별 | 8요소 리스크 가정 매핑 | 가정 리스트 |
| 4. 가정 우선순위 | Impact × Risk 매트릭스 | 우선순위 가정 |
| 5. 실험 설계 | 최소 비용 검증 실험 | 실험 계획 |

### 실습

```
/discover 와키윌리 웰니스 라인 진출 기회
```

## 3. `/brainstorm` — 멀티 관점 브레인스토밍

### 사용법

```
/brainstorm [주제]
```

### 4가지 관점에서 아이디어 발산

| 관점 | 질문 |
|------|------|
| **고객 관점** | 고객이 해결 못 하고 있는 문제는? |
| **경쟁 관점** | 경쟁사가 안 하고 있는 것은? |
| **기술 관점** | 새로운 기술로 가능해진 것은? |
| **트렌드 관점** | 지금 떠오르는 트렌드와 연결할 수 있는 것은? |

### 아이디어 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **신규 상품** | 완전히 새로운 카테고리 | 와키윌리 웰니스 라인 |
| **기존 개선** | 현재 상품/서비스 개선 | 캠프키치 소재 업그레이드 |
| **프로세스 혁신** | 일하는 방식 개선 | QR 리드타임 50% 단축 |
| **경험 혁신** | 고객 경험 개선 | 매장 피팅룸 디지털화 |

### 실습

```
/brainstorm 와키윌리 26FW 시즌 테마 아이디어
```

## 4. 가정 식별 & 우선순위

### 가정 식별 — 8요소 리스크 매핑

```
"이 아이디어에서 위험한 가정이 뭐야?"
```

| # | 가정 영역 | 질문 |
|---|----------|------|
| 1 | **고객 가치** | 고객이 진짜 원하는 건가? |
| 2 | **사용성** | 고객이 쉽게 이해하고 사용할 수 있는가? |
| 3 | **실현 가능성** | 우리가 만들 수 있는가? |
| 4 | **비즈니스 모델** | 돈이 되는가? |
| 5 | **채널** | 고객에게 도달할 수 있는가? |
| 6 | **시장 규모** | 충분히 큰 시장인가? |
| 7 | **경쟁** | 경쟁사보다 나은가? |
| 8 | **타이밍** | 지금이 적절한 시점인가? |

### 가정 우선순위 — Impact × Risk 매트릭스

```
"어떤 가정부터 검증해야 해?"
```

```
        High Impact
   ┌─────────┬─────────┐
   │ 모니터  │ 즉시    │
   │         │ 검증!   │ ← 여기부터
   ├─────────┼─────────┤
   │ 무시    │ 나중    │
   │         │ 검증    │
   └─────────┴─────────┘
  Low Risk        High Risk
```

## 5. 실험 설계

### XYZ 가설 형식

```
[X]를 하면
[Y]가 [Z]만큼 변할 것이다

예시:
"인스타에 웰니스 라인 티저를 올리면,
 저장 수가 일반 포스팅 대비 2배 이상 나올 것이다"
```

### 실험 유형

| 유형 | 비용 | 시간 | 적합한 경우 |
|------|------|------|-----------|
| **고객 인터뷰** | 매우 낮음 | 1~2주 | 고객 가치 가정 검증 |
| **랜딩 페이지** | 낮음 | 1주 | 수요 검증 |
| **SNS 테스트** | 낮음 | 3일 | 메시지·비주얼 반응 검증 |
| **소량 생산** | 중간 | 4~6주 | 상품 적합성 검증 |
| **팝업 스토어** | 중간 | 2~4주 | 오프라인 반응 검증 |

## 6. `/triage` — 기능 요청 트리아지

고객·바이어·내부에서 들어온 요청을 정리하고 우선순위를 매깁니다.

```
/triage [요청 리스트]
```

### 트리아지 기준

| 기준 | 설명 |
|------|------|
| **빈도** | 얼마나 많은 사람이 요청하는가 |
| **영향** | 해결하면 비즈니스에 얼마나 도움이 되는가 |
| **긴급도** | 지금 안 하면 어떤 문제가 생기는가 |
| **실현 가능성** | 우리가 할 수 있는가 |

## 7. `/pre-mortem` — 런칭 전 리스크 점검

```
/pre-mortem [프로젝트명]
```

**"이 프로젝트가 실패한다면, 왜 실패할까?"**를 사전에 시뮬레이션.

### 리스크 3분류

| 유형 | 설명 | 대응 |
|------|------|------|
| **Tigers** | 확실히 위험한 것 | 즉시 완화 계획 |
| **Paper Tigers** | 위험해 보이지만 실제로는 아닌 것 | 모니터링만 |
| **Elephants** | 모두 알지만 아무도 말하지 않는 것 | 공론화 + 대응 |

### Go/No-Go 판단

리스크 분석 후 최종 판단:
- **Go**: 리스크 완화 계획과 함께 진행
- **Conditional Go**: 특정 조건 충족 시 진행
- **No-Go**: 중단 또는 피벗

### 실습

```
/pre-mortem TRUSTED COMFORT 26FW 런칭
```

## 8. 전체 디스커버리 → 실행 파이프라인

```
/discover → 기회 탐색 + 아이디어 + 가정 + 실험
    ↓
/pre-mortem → 런칭 전 리스크 점검
    ↓
/prd → 요구사항 정의서 작성
    ↓
/okrs → 목표 설정
    ↓
/sprint plan → 스프린트 계획
    ↓
실행 → /sprint retro → 회고 → 반복
```
