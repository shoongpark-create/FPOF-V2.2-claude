# 세션 7: 프레젠테이션 자료 만들기

> **소요시간**: 35분 (설명 10분 + 데모 15분 + 실습 10분)
> **대상**: 전 멤버
> **목표**: PPTX 파일과 HTML 프레젠테이션을 AI로 자동 생성하는 3가지 방식을 이해하고, 상황에 맞는 도구를 선택

---

## 1. 프레젠테이션 제작 — 3가지 방식 비교

FPOF에서 프레젠테이션을 만드는 방식은 **3가지**이며, 용도가 다릅니다.

| 방식 | 출력 형식 | 용도 | 특징 |
|------|----------|------|------|
| **`/deck`** | `.pptx` | FPOF 브랜드 덱 (시즌별 표준) | 와키윌리 컬러·톤 자동 적용, PDCA 단계별 덱 유형 |
| **PPTX 스킬** | `.pptx` | 범용 파워포인트 | 디자인 자유도 높음, 템플릿 편집·신규 생성 모두 가능 |
| **Frontend Slides** | `.html` | 웹 프레젠테이션 | 애니메이션, 반응형, 브라우저에서 바로 발표, PPT 변환도 가능 |

### 언제 무엇을 쓰나?

```
"바이어 미팅용 시즌 덱 만들어줘"
  → /deck buyer  (FPOF 표준 + 와키윌리 브랜드)

"이 데이터로 프레젠테이션 만들어줘"
  → PPTX 스킬  (범용, 자유로운 디자인)

"임원 보고용으로 멋진 웹 프레젠테이션 만들어줘"
  → Frontend Slides  (애니메이션, 인터랙티브)

"이 PPT를 웹으로 변환해줘"
  → Frontend Slides  (PPT → HTML 변환)
```

## 2. 방식 1: `/deck` — FPOF 브랜드 덱

### 기본 사용법

```
/deck [유형]
```

유형을 생략하면 현재 PDCA 단계에 맞는 덱을 추천합니다.

### PDCA 단계별 덱 유형

| 단계 | 유형 | 내용 |
|------|------|------|
| **Plan** | `trend` | 트렌드 리서치 데크 — 글로벌/로컬 트렌드, 키 비주얼, 시사점 |
| | `season` | 시즌 프레젠테이션 — 테마, 컨셉, 컬러, 무드, 전략 |
| | `md-plan` | MD 전략 데크 — 카테고리 믹스, 챔피언 상품, 가격 전략 |
| **Design** | `moodboard` | 무드보드 데크 — 비주얼 톤, 컬러 팔레트, 레퍼런스 |
| | `lookbook` | 룩북 프레젠테이션 — 디자인 스케치, 스타일링 가이드 |
| **Do** | `buyer` | 바이어 프레젠테이션 — 라인시트, 사이즈런, 가격표 |
| | `campaign` | 캠페인 데크 — IMC 전략, 채널 플랜, 타임라인 |
| **Check** | `report` | 시즌 리포트 데크 — 매출 분석, KPI 대시보드, 인사이트 |

### 자동 적용 요소

| 요소 | 설명 |
|------|------|
| **브랜드 컬러** | `visual-identity.json` 기반 (Primary #FF6B35, Secondary #2EC4B6 등) |
| **톤앤매너** | `tone-manner.json` 기반 와키윌리 브랜드 보이스 |
| **슬라이드 구성** | 표지 → 목차 → 본문 → 요약 → 부록 (표준) |
| **IP 캐릭터** | 키키(Kiki) 등 활용 가능 시 포함 |

### 사용 예시

```
# 시즌 프레젠테이션
/deck season

# 바이어 프레젠테이션
/deck buyer

# 유형 생략 시 → 현재 PDCA 단계에 맞는 추천
/deck
```

### 산출물 저장

```
workspace/26SS/[프로젝트]/deck_[유형]_[날짜].pptx
```

## 3. 방식 2: PPTX 스킬 — 범용 파워포인트

### 3가지 작업 모드

| 모드 | 설명 | 명령 |
|------|------|------|
| **읽기/분석** | 기존 PPTX 내용 추출·분석 | "이 PPT 내용 정리해줘" |
| **템플릿 편집** | 기존 PPT 기반으로 수정·업데이트 | "이 PPT에서 3페이지 수정해줘" |
| **새로 만들기** | 처음부터 새 프레젠테이션 생성 | "프레젠테이션 만들어줘" |

### 디자인 원칙 (PPTX 스킬 내장)

스킬이 자동으로 지키는 디자인 규칙:

| 원칙 | 설명 |
|------|------|
| **주제에 맞는 컬러** | 범용 파란색 금지. 주제에 어울리는 팔레트 선택 |
| **컬러 비중 6:3:1** | 주색 60%, 보조색 30%, 강조색 10% |
| **매 슬라이드 비주얼** | 텍스트만 있는 슬라이드 금지. 이미지·차트·아이콘 필수 |
| **레이아웃 변화** | 같은 레이아웃 반복 금지. 컬럼·카드·콜아웃 다양하게 |
| **본문 좌측 정렬** | 제목만 가운데, 본문은 좌측 정렬 |
| **여백 확보** | 최소 0.5인치 마진, 콘텐츠 사이 0.3~0.5인치 간격 |

### 내장 컬러 팔레트 (영감용)

| 테마 | Primary | Secondary | Accent | 적합한 용도 |
|------|---------|-----------|--------|------------|
| Midnight Executive | Navy `1E2761` | Ice Blue `CADCFC` | White | 경영진 보고 |
| Coral Energy | Coral `F96167` | Gold `F9E795` | Navy | 마케팅·캠페인 |
| Warm Terracotta | Terracotta `B85042` | Sand `E7E8D1` | Sage | 소재·무드보드 |
| Forest & Moss | Forest `2C5F2D` | Moss `97BC62` | Cream | 웰니스·자연 컨셉 |
| Berry & Cream | Berry `6D2E46` | Dusty Rose `A26769` | Cream | 우먼스·프리미엄 |
| Cherry Bold | Cherry `990011` | Off-White `FCF6F5` | Navy | 임팩트·런칭 |

### 사용 예시

```
# 새 프레젠테이션
"마케팅 전략 프레젠테이션 만들어줘. Coral Energy 컬러로."

# 기존 PPT 수정
"이 PPT 5페이지 차트 데이터를 업데이트해줘"
→ 파일 경로 전달

# PPT 내용 추출
"이 PPT 내용 마크다운으로 정리해줘"
```

### QA (자동 품질 검수)

PPTX 스킬은 생성 후 **자동으로 QA를 실행**합니다:

1. **콘텐츠 QA** — 누락 텍스트, 오타, 순서 확인
2. **비주얼 QA** — 겹치는 요소, 텍스트 잘림, 간격 문제 감지
3. **플레이스홀더 QA** — 템플릿 남은 텍스트(XXXX, Lorem 등) 검출

## 4. 방식 3: Frontend Slides — 웹 프레젠테이션

### 특징

| 항목 | 설명 |
|------|------|
| **출력** | 단일 `.html` 파일 (CSS/JS 인라인, 의존성 없음) |
| **애니메이션** | 슬라이드 전환, 요소 등장, 스크롤 효과 |
| **반응형** | 데스크톱·태블릿·모바일 자동 대응 |
| **발표** | 브라우저에서 바로 풀스크린 발표 가능 |
| **PPT 변환** | 기존 PPTX → HTML 변환 지원 |
| **라이트 모드 기본** | 프로젝터 호환성·가독성 우선 (다크 모드는 요청 시) |

### 3가지 모드

| 모드 | 설명 |
|------|------|
| **새로 만들기** | 주제·스타일 선택 → HTML 프레젠테이션 생성 |
| **PPT 변환** | 기존 `.pptx` 파일을 웹 프레젠테이션으로 변환 |
| **기존 개선** | 기존 HTML 프레젠테이션 디자인 개선 |

### 스타일 프리셋 12종

"어떤 느낌으로 만들까?"를 선택하면 됩니다:

| 프리셋 | 느낌 | 추천 용도 |
|--------|------|----------|
| **Bold Signal** | 자신감, 임팩트 | 피치 덱, 키노트 |
| **Electric Studio** | 깔끔, 프로페셔널 | 에이전시·클라이언트 발표 |
| **Creative Voltage** | 에너제틱, 레트로모던 | 크리에이티브 제안 |
| **Dark Botanical** | 우아, 세련 | 프리미엄 브랜드 |
| **Notebook Tabs** | 에디토리얼, 정돈 | 리포트, 리뷰 |
| **Pastel Geometry** | 친근, 접근성 | 상품 소개 |
| **Split Pastel** | 플레이풀, 모던 | 크리에이티브 에이전시 |
| **Vintage Editorial** | 위트, 개성 | 퍼스널 브랜드 |
| **Neon Cyber** | 미래지향, 테크 | 테크 스타트업 |
| **Terminal Green** | 개발자 감성 | 기술 프레젠테이션 |
| **Swiss Modern** | 미니멀, 정밀 | 기업·데이터 |
| **Paper & Ink** | 문학적, 사려 깊은 | 스토리텔링 |

### 스타일 선택 방식 2가지

**방식 A: 가이드 디스커버리 (추천)**
- AI가 분위기 질문 → 3개 프리뷰 생성 → 마음에 드는 것 선택
- "어떤 스타일이 좋을지 모르겠어"라는 분에게 추천

**방식 B: 직접 선택**
- "Bold Signal 스타일로 만들어줘"처럼 프리셋명 직접 지정

### 슬라이드 밀도 제한 (뷰포트 피팅)

웹 프레젠테이션은 **스크롤 없이** 한 화면에 모든 내용이 보여야 합니다:

| 슬라이드 유형 | 최대 콘텐츠 |
|-------------|------------|
| 타이틀 | 제목 1개 + 부제 1개 + 태그라인 (선택) |
| 콘텐츠 | 제목 1개 + 불릿 4~6개 또는 문단 2개 |
| 그리드 | 제목 1개 + 카드 최대 6개 (2×3 또는 3×2) |
| 인용 | 인용문 3줄 이내 + 출처 |
| 이미지 | 제목 1개 + 이미지 1개 (최대 60vh) |

> 내용이 넘치면? → **슬라이드를 분할** (폰트 축소 금지)

### 사용 예시

```
# 새 프레젠테이션
"TRUSTED COMFORT 전략 발표용 웹 프레젠테이션 만들어줘"

# PPT → HTML 변환
"이 PPT 파일을 웹 프레젠테이션으로 변환해줘"
→ 파일 경로 전달

# 스타일 직접 지정
"Bold Signal 스타일로 IMC 전략 발표 프레젠테이션 만들어줘"
```

## 6. 워크플로우 — 다른 스킬과의 연계

### 보고서 → 프레젠테이션

```
[Step 1] /meeting 으로 회의록 생성
    ↓
[Step 2] "임원 보고용으로 요약해줘" → executive-summary
    ↓
[Step 3] "이걸 프레젠테이션으로 만들어줘" → /deck 또는 Frontend Slides
```

### 분석 → 프레젠테이션

```
[Step 1] 매출 데이터 분석 (데이터 인텔리전스)
    ↓
[Step 2] "분석 결과를 프레젠테이션으로 만들어줘" → /deck report
```

### PPTX ↔ HTML 변환

```
# PPTX → HTML (웹 발표용)
"이 PPT를 웹 프레젠테이션으로 변환해줘"

# HTML → PPTX (오프라인 발표·인쇄용)
→ 현재 직접 지원 안 됨. HTML을 참고하여 /deck으로 새로 생성
```

## 7. 디자인 팁 — 좋은 프레젠테이션의 원칙

### DO (이렇게)

| 원칙 | 설명 |
|------|------|
| **1슬라이드 = 1메시지** | 한 화면에 하나의 핵심만 전달 |
| **비주얼 우선** | 텍스트보다 이미지·차트·아이콘이 먼저 눈에 들어오게 |
| **여백 확보** | 빈 공간은 낭비가 아니라 가독성 |
| **대비 강하게** | 제목 36pt+ / 본문 14~16pt — 크기 차이 확실하게 |
| **브랜드 컬러 일관** | 와키윌리 덱이면 visual-identity.json 컬러 필수 |
| **다크/라이트 샌드위치** | 표지·결론 = 다크 배경, 본문 = 라이트 배경 |

### DON'T (이렇게 하지 마세요)

| 실수 | 대안 |
|------|------|
| ❌ 같은 레이아웃 반복 | 컬럼·카드·콜아웃 등 다양하게 |
| ❌ 텍스트만 있는 슬라이드 | 아이콘·차트·이미지 최소 1개 포함 |
| ❌ 범용 파란색 디폴트 | 주제에 맞는 컬러 팔레트 선택 |
| ❌ 본문 가운데 정렬 | 제목만 가운데, 본문은 좌측 정렬 |
| ❌ 제목 아래 악센트 라인 | AI 생성 느낌. 여백이나 배경색으로 대체 |
| ❌ 폰트 축소해서 우겨넣기 | 슬라이드 분할이 정답 |

## 9. 결정 가이드 — 어떤 방식을 선택할까?

| 상황 | 추천 방식 | 이유 |
|------|----------|------|
| 바이어 미팅·경영진 보고 (오프라인 발표) | `/deck` | PPTX 파일, 브랜드 자동 적용 |
| 시즌 룩북·무드보드 (인쇄·공유) | `/deck` | PPTX, 이미지 삽입 용이 |
