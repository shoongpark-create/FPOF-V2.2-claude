# 세션 6: 회의록 정리 & 보고서 요약 스킬

> **소요시간**: 30분 (설명 10분 + 데모 10분 + 실습 10분)
> **대상**: 전 멤버
> **목표**: 회의 트랜스크립트를 구조화된 회의록으로 변환하고, 경영진 보고용 요약 보고서를 자동 생성하는 워크플로우 실습

---

## 1. 왜 회의록 스킬이 필요한가?

### 현재 문제

| 문제 | 영향 |
|------|------|
| 회의 후 정리에 **30분~1시간** 소요 | 실무 시간 잠식 |
| 결정 사항·액션 아이템이 **흩어짐** | 후속 조치 누락 |
| 사람마다 **다른 형식**으로 기록 | 검색·공유 어려움 |
| 경영진 보고용으로 **다시 가공** 필요 | 이중 작업 |

### AI 스킬 적용 후

```
회의 트랜스크립트 (Otter, Clova, Zoom, 수기 메모)
    ↓
[/meeting] 구조화된 회의록 자동 생성 (5분)
    ↓
[executive-summary] 경영진 보고용 요약 자동 변환 (2분)
```

**결과**: 1시간 → 7분으로 단축. 표준 형식 보장, 누락 자동 감지.

---

## 2. 스킬 구성 — 2개 스킬의 역할 분담

FPOF에는 회의록 관련 **2개 스킬**이 있으며, 용도가 다릅니다.

| 스킬 | 슬래시 명령 | 용도 | 출력 형식 |
|------|-----------|------|----------|
| **회의록 작성** | `/meeting` | 회의 트랜스크립트 → 구조화된 회의록 | 참석자·결정사항·액션아이템·미해결사항 |
| **보고서 요약** | 자동 트리거 | 회의록/리포트 → 경영진 보고용 요약 | Summary·Key Metrics·Insights·Action Items |

### 사용 흐름

```
[상황 1] 회의 직후 정리
→ /meeting 사용 → 구조화된 회의록 생성

[상황 2] 회의록을 경영진에게 보고
→ "이 회의록 임원 보고용으로 정리해줘" → executive-summary 자동 트리거

[상황 3] 여러 회의·리포트를 합쳐서 보고
→ "이번 주 회의 3건 합쳐서 주간 보고 만들어줘" → executive-summary 자동 트리거

[상황 4] 마케팅 리포트·Slack 로그 등 비회의 텍스트 요약
→ "이 캠페인 결과 보고용으로 요약해줘" → executive-summary 자동 트리거
```

## 3. 스킬 1: `/meeting` — 회의록 작성

### 기본 사용법

```
/meeting <회의 내용 붙여넣기>
```

### 지원 입력 형식

| 형식 | 예시 |
|------|------|
| **전체 트랜스크립트** | Otter, Clova, Zoom 자동 생성 텍스트 |
| **수기 메모** | 직접 작성한 회의 노트 |
| **오디오 요약** | 녹음 파일의 AI 요약본 |
| **혼합 입력** | 트랜스크립트 + 개인 메모 |
| **파일 첨부** | `.txt`, `.md`, `.docx` 파일 경로 전달 |

### 출력 형식

```markdown
# 회의 요약

**일시**: 2026-03-25 14:00~15:30
**참석자**: 김브랜드(브랜드팀장), 이마케팅(마케팅), 박MD(MD팀)
**주제**: 26SS IMC 캠페인 1차 리뷰

## 요약
- **포인트 1**: 주력 스타일 3종 선정 — 캠프키치 후드티, 트러스티드 맨투맨, IP 그래픽 티
- **포인트 2**: 인플루언서 시딩 1차 결과 CTR 3.8%, 목표(3.5%) 달성
- **포인트 3**: 2차 시딩은 마이크로 인플루언서 중심으로 전환 결정

## 결정 사항
- 주력 스타일은 3종으로 확정, 추가 1종은 QR 결과 보고 후 결정
- 2차 시딩 예산 500만원 → 마이크로 인플루언서 20명으로 분산 집행
- 무신사 단독 상품 기획안은 다음 주 월요일까지 제출

## 액션 아이템

| 기한 | 담당자 | 액션 |
|------|--------|------|
| 2026-03-28 | 이마케팅 | 마이크로 인플루언서 20명 리스트업 |
| 2026-03-28 | 박MD | 무신사 단독 상품 기획안 작성 |
| 2026-03-31 | 김브랜드 | 2차 시딩 크리에이티브 브리프 승인 |
| 2026-04-01 | 이마케팅 | 1차 시딩 성과 리포트 최종본 제출 |

## 미해결 사항
- QR 추가 물량 확정 시점 (공장 리드타임 확인 필요)
- 셀럽 협업 건은 법무 검토 완료 후 재논의

## 시즌 산출물 영향
- `do_imc-strategy.md` 2차 시딩 전략 업데이트 필요
- `do_social-viral.md` 인플루언서 리스트 반영
```

### 패션 업계 특화 기능

| 기능 | 설명 |
|------|------|
| **시즌 마일스톤 연결** | 액션 아이템 기한을 시즌 일정과 연결 |
| **FPOF 팀 매핑** | 담당자를 에이전시·역할에 자동 매핑 |
| **산출물 영향 표시** | 결정 사항이 영향을 미치는 산출물 파일 식별 |
| **PDCA 단계 반영** | 현재 단계(Plan/Design/Do/Check/Act)에 맞는 맥락 제공 |

## 4. 스킬 2: Executive Summary — 보고서 요약

### 자동 트리거 키워드

아래 키워드가 포함된 요청에서 **자동으로 작동**합니다:

| 키워드 | 사용 예시 |
|--------|----------|
| `주간 보고` | "이 회의록 주간 보고용으로 정리해줘" |
| `임원 보고` | "임원 보고용 요약으로 만들어줘" |
| `캠페인 결과 요약` | "이번 캠페인 결과 요약해줘" |
| `프로젝트 회의 요약` | "프로젝트 회의 요약 부탁해" |
| `보고용으로 요약` | "이 내용 보고용으로 요약해줘" |
| `보고서 작성` | "마케팅 보고서 작성해줘" |
| `회의록 정리` | "회의록 정리해줘" (회의 트랜스크립트가 아닌 기존 회의록 재정리 시) |

### 작업 4단계

```
1단계: 정보 추출 (Extraction)
→ 핵심 목표, 수치/지표, 결정사항, 이슈, 액션아이템, 미결사항

2단계: 구조화 재정리 (Structuring)
→ Summary · Key Metrics · Insights · Action Items · 추가 필요 정보

3단계: 문체 적용 (Styling)
→ 경어체, 한영 병기, 숫자 천단위 쉼표, YYYY-MM-DD

4단계: 누락 확인 (Validation)
→ 담당자 없는 액션, 마감일 없는 할 일, 비교 기준 없는 수치 자동 감지
```

### 출력 형식

```markdown
# 26SS IMC 캠페인 1차 리뷰 요약

> **작성일:** 2026-03-25
> **작성자:** AI 자동 생성
> **보고 대상:** 주간 경영회의

---

## Summary

1. IMC 주력 스타일 3종이 확정되었으며, 1차 인플루언서 시딩 CTR 3.8%로 목표를 달성했습니다.
2. 2차 시딩은 마이크로 인플루언서 20명으로 전환하여 비용 효율을 높이기로 결정했습니다.
3. 무신사 단독 상품 기획안은 3/28까지 제출하며, QR 추가 물량은 공장 확인 후 결정합니다.

---

## Key Metrics

| 지표 | 값 | 전기 대비 | 비고 |
|------|-----|----------|------|
| 1차 시딩 CTR | 3.8% | +0.3%p | 목표(3.5%) 달성 |
| 2차 시딩 예산 | 500만원 | - | 마이크로 20명 분산 |
| 주력 스타일 | 3종 | - | 캠프키치·트러스티드·IP그래픽 |

---

## Insights

### 성과 요인
- 1차 시딩에서 숏폼 영상 콘텐츠가 정적 이미지 대비 CTR +1.2%p 높은 효과를 보였습니다.

### 개선 필요
- 파워 인플루언서 단가 대비 전환 효율이 낮아, 마이크로 인플루언서로 전환이 적절합니다.

---

## Action Items

### 즉시 실행 (이번 주)
- [ ] 마이크로 인플루언서 20명 리스트업 | 담당: 이마케팅 | 마감: 2026-03-28
- [ ] 무신사 단독 상품 기획안 작성 | 담당: 박MD | 마감: 2026-03-28

### 단기 계획 (2주 내)
- [ ] 2차 시딩 크리에이티브 브리프 승인 | 담당: 김브랜드 | 마감: 2026-03-31
- [ ] 1차 시딩 성과 리포트 최종본 | 담당: 이마케팅 | 마감: 2026-04-01

---

## 추가로 필요한 정보

1. QR 추가 물량 확정 시점 — 공장 리드타임 확인 필요
2. 셀럽 협업 건 법무 검토 완료 예상일은?
```

### 입력 유형별 처리

| 입력 유형 | 주요 추출 포인트 |
|-----------|-----------------|
| **회의록** | 결정사항, 논의 내용, 액션 아이템, 참석자 |
| **이메일 스레드** | 핵심 요청, 승인 사항, 마감일, 담당자 |
| **마케팅 리포트** | KPI 수치, 캠페인 성과, 개선점, 예산 |
| **Slack/Teams 로그** | 결론, 합의사항, 멘션된 담당자, 기한 |
| **혼합 입력** | 여러 소스를 합쳐서 통합 요약 |

## 5. 2개 스킬 연계 워크플로우

### 워크플로우 A: 회의 → 회의록 → 보고서 (풀 파이프라인)

```
[Step 1] 회의 끝나고 트랜스크립트 확보
         (Otter/Clova에서 복사 또는 수기 메모)
    ↓
[Step 2] /meeting 으로 구조화된 회의록 생성
         → workspace/26SS/weekly/w13/meeting_imc-review_2026-03-25.md
    ↓
[Step 3] "이 회의록 임원 보고용으로 정리해줘"
         → executive-summary가 자동 트리거
         → Summary·Metrics·Insights·Action Items 포맷으로 변환
    ↓
[Step 4] 필요시 PPTX/PDF로 내보내기
         → /deck 또는 /pdf 명령어 활용
```

### 워크플로우 B: 다건 회의 통합 주간 보고

```
[Step 1] 이번 주 회의록 3건 확보
         - meeting_imc-review.md
         - meeting_md-planning.md
         - meeting_buyer-presentation.md
    ↓
[Step 2] "이번 주 회의 3건 합쳐서 주간 보고 만들어줘"
         → executive-summary가 3건을 통합 요약
         → 공통 결정사항·액션 아이템을 머지하여 1건 보고서 생성
    ↓
[Step 3] 주간 리뷰 산출물로 저장
         → workspace/26SS/weekly/w13/review_exec-summary_2026-03-25.md
```

### 워크플로우 C: 비회의 텍스트 보고서 변환

```
[Step 1] 캠페인 성과 데이터, Slack 대화, 이메일 스레드 등 확보
    ↓
[Step 2] "이 캠페인 결과 보고용으로 요약해줘"
         → executive-summary 자동 트리거
         → 표준 보고서 형식으로 변환
```

## 6. 품질 기준 — 좋은 보고서 vs 나쁜 보고서

### 좋은 보고서 (GOOD)

| 항목 | 기준 |
|------|------|
| Summary | 3줄 이내, 목적-결과-다음단계 구조 |
| Key Metrics | 수치가 표로 정리, 전기/목표 대비 포함 |
| Insights | "왜"에 대한 원인 분석 포함 |
| Action Items | 담당자 + 마감일 **모두** 명시 |
| 압축률 | 원문 대비 70% 이상 압축 |

### 나쁜 보고서 (BAD)

```
❌ "이번 주 캠페인이 잘 되었습니다. 클릭률이 올랐고 전환도 좋아졌습니다."
   → 수치 없음, 구조 없음, 모호한 표현

❌ 할 일: 소재 만들기, 테스트 하기, 보고서 쓰기
   → 담당자 없음, 마감일 없음, 추적 불가
```

### 자동 감지 — 누락 정보 경고

스킬이 자동으로 감지하여 `## 추가로 필요한 정보`에 질문을 생성합니다:

| 누락 유형 | 자동 질문 예시 |
|-----------|-------------|
| 담당자 없는 액션 | "신규 캠페인 A/B 테스트의 담당자는 누구인가요?" |
| 마감일 없는 할 일 | "인플루언서 리스트업의 기한은 언제인가요?" |
| 비교 기준 없는 수치 | "CTR 3.2%의 비교 기준(전기/목표치)은 무엇인가요?" |
| 맥락 없는 언급 | "Q1 마감 언급이 있는데, 정확한 날짜는 언제인가요?" |

## 7. 실습

### 실습 1: `/meeting` 회의록 생성 (5분)

아래 샘플 트랜스크립트를 Claude Code에 붙여넣고 `/meeting`을 실행해 보세요:

```
/meeting

[샘플 트랜스크립트]
참석: 김팀장, 이대리, 박사원
일시: 3월 25일 오후 2시

김팀장: 이번 주 매출 현황 먼저 공유해주세요.
이대리: 네, 이번 주 와키윌리 전체 매출 3억 2천만원입니다.
전주 대비 12% 증가했고, 특히 캠프키치 후드티가 누적 2,300장
판매되면서 시즌 1위를 유지하고 있습니다.
박사원: 무신사 채널은 전주 대비 -5%인데요, 프로모션 기간이
끝나면서 자연 감소한 것 같습니다.
김팀장: 무신사는 인플루언서 콜라보로 다시 끌어올려야 합니다.
다음 주까지 인플루언서 리스트 만들어주세요.
이대리: 넵, 마이크로 인플루언서 위주로 20명 리스트업 하겠습니다.
김팀장: 그리고 캠프키치 리오더 건은 어떻게 됐어요?
박사원: 공장에서 4월 10일 입고 가능하다고 합니다. 500장 추가 발주
하면 될 것 같습니다.
김팀장: 좋습니다. 500장 발주 진행하세요. 이번 주 금요일까지 발주서
완료 부탁합니다.
```

**확인 포인트:**
- [ ] 참석자와 역할이 정리되었는가?
- [ ] 결정 사항(인플루언서 콜라보, 리오더 500장)이 분리되었는가?
- [ ] 액션 아이템에 담당자와 기한이 있는가?
- [ ] 미해결 사항이 별도로 정리되었는가?

### 실습 2: Executive Summary 변환 (3분)

실습 1에서 만든 회의록을 경영진 보고용으로 변환해 보세요:

```
이 회의록 임원 보고용으로 정리해줘
```

**확인 포인트:**
- [ ] Summary가 3줄 이내인가?
- [ ] Key Metrics 표에 수치가 정리되었는가?
- [ ] Insights에 원인 분석이 있는가?
- [ ] Action Items에 담당자·마감일이 모두 있는가?

### 실습 3: 자신의 실제 회의록 변환 (선택)

| AOV | Average Order Value | 평균 주문 금액 |
| ASP | Average Selling Price | 평균 판매 단가 |
| STR | Sell-Through Rate | 소진율 |
| WoW | Week over Week | 전주 대비 |

## 9. 표기 규칙

보고서에서 일관된 표기를 위해 아래 규칙을 따릅니다:

| 규칙 | 예시 |
|------|------|
| **경어체** 사용 | "~입니다", "~해 주세요" |
| **한영 병기** | "전환율(Conversion Rate)" |
| **숫자 표기** | 천 단위 쉼표: "1,234,567원" |
| **날짜 형식** | `YYYY-MM-DD` (예: 2026-03-25) |
| **백분율** | 소수점 1자리: "12.5%" |
| **액션 아이템** | `- [ ] 할 일 \| 담당: 이름 \| 마감: YYYY-MM-DD` |
