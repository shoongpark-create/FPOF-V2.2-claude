## 1. PMF란 무엇인가

### 한 문장 정의

> **"우리 상품이 고객의 진짜 니즈를 충족시켜서, 고객이 알아서 찾아오고 재구매하는 상태"**

PMF는 마크 앤드리센(Marc Andreessen)이 만든 개념으로, 원래 테크 스타트업에서 쓰이지만 패션에서도 핵심 원리는 동일합니다.

### 패션에서의 PMF 공식

```
PMF = 맞는 고객(Market) × 맞는 상품(Product) × 맞는 채널(Channel)
```

- **Market** → 누가 사는가 (타겟 세그먼트)
- **Product** → 왜 사는가 (가치 제안)
- **Channel** → 어디서 만나는가 (유통·마케팅 접점)

세 가지가 모두 맞물려야 PMF가 달성됩니다. 하나라도 빠지면:

| 빠진 요소 | 증상 |
|-----------|------|
| Market ✕ | 좋은 상품인데 살 사람이 없다 — "예쁜데 안 팔려" |
| Product ✕ | 고객은 있는데 맞는 상품이 없다 — "와키윌리에서 이걸 왜?" |
| Channel ✕ | 맞는 상품인데 고객이 발견 못한다 — "이런 게 있는 줄 몰랐어" |

### PMF가 있으면 느낌으로 안다

| PMF 달성 전 | PMF 달성 후 |
|------------|------------|
| "이거 사줄 수 있어요?" 가 필요 | 고객이 먼저 찾는다 |
| 재고가 할인으로 소진 | 정상가 판매율 70%+ |
| SNS 반응이 광고에만 의존 | 고객이 자발적으로 공유 |
| 리오더 없이 시즌 종료 | QR 리오더 2-3차 발생 |
| 구매 후 재구매 없음 | "다른 색도 사야지" 재구매 |

## 2. 왜 패션 브랜드에 PMF가 중요한가

### 와키윌리의 현실적 도전

우리 브랜드 데이터에서 핵심 괴리가 발견됩니다:

```
검색(인지) → 40대 여성 63%, 여성 83%
오프라인 구매 → 20대 19%, 25대 18% (코어 타겟 37%)
온라인 구매 → 40대 25%, 20대 23%
```

**해석**: 검색(관심)하는 사람과 실제 구매하는 사람이 다릅니다. 이것은 PMF가 완전하지 않다는 신호입니다.

### PMF가 해결하는 5대 경영목표와의 연결

| 경영목표 | PMF 기여 |
|---------|---------|
| ① 브랜드 아이덴티티 정립 | 코어 타겟(18-25)이 "이게 내 브랜드"라고 느끼는 상태 |
| ② 히트상품 + IMC 강화 | 상위 20%가 매출 50% 리딩 = PMF 달성된 상품 |
| ③ QR 비중 확대 | 리오더가 나온다 = 고객이 더 원한다 = PMF 시그널 |
| ④ 용품 경쟁력 확보 | 용품도 고객의 Job-to-be-Done이 있어야 팔린다 |
| ⑤ 글로벌 대응 강화 | 시장마다 PMF를 다시 검증해야 한다 (한국 PMF ≠ 대만 PMF) |

## 3. STAGE 1 — 시장-고객 정의

> **핵심 질문**: "정확히 누가, 어떤 상황에서, 무엇 때문에 우리 옷을 사는가?"

### 3-1. 왜 "20대 여성"으로는 부족한가

"타겟: 18-25세 트렌드리더"는 시작점이지 결론이 아닙니다. PMF를 위해서는 **상황(Situation) 기반 세분화**가 필요합니다.

같은 22세 여대생이라도:

| 상황 | 원하는 것 | 선택 기준 |
|------|----------|----------|
| 대학 등교 | 편하고 센스 있는 데일리 | 가격 접근성, 코디 용이성 |
| 친구 약속 (홍대) | 눈에 띄는 포인트 아이템 | 유니크함, SNS 공유 가치 |
| 필라테스 → 카페 | 갈아입지 않아도 되는 옷 | 기능+감도 겸비, 멀티씬 |
| 팝업스토어 방문 | 한정판·경험 소비 | 희소성, 브랜드 세계관 |

**같은 사람이 상황에 따라 다른 상품을 원합니다.**

### 3-2. FPOF 스킬로 시장-고객 정의하기

#### 🔍 STEP 1: 거시 환경 스캔

**사용 스킬**: `/market-scan` (거시 환경 분석)

무엇을 분석하는가:
- PESTLE 분석 — 정치·경제·사회·기술·법률·환경 트렌드
- 패션 시장 매크로 동향 (시장 규모, 성장률, 세그먼트 변화)

**와키윌리 사례 — TRUSTED COMFORT 프로젝트**:

```
거시 발견: 글로벌 라운지웨어 시장 $129억(2024) → $326억(2033), CAGR 7.9%
사회 트렌드: 소비자 54%가 실내외 겸용 멀티펑셔널 라운지웨어 선호
시사점: "편안함"이 더 이상 집에서만의 가치가 아님 → 멀티씬 웨어 기회
```

#### 🔍 STEP 2: 트렌드 레이더

**사용 스킬**: `/trend-radar` (트렌드 레이더)

무엇을 분석하는가:
- 현재 시장에서 뜨고 있는 키워드·실루엣·소재·컬러
- 경쟁 브랜드 동향 (무신사 랭킹, 글로벌 레퍼런스)
- 1~2년 내 확산 가능한 트렌드 vs 이미 정점을 지난 트렌드

**실행 방법**:
```
"/trend-radar" 실행 → 자동으로 다음을 수행:
1. 무신사 랭킹/발매 데이터 수집
2. 글로벌 트렌드 리포트 분석
3. SNS 키워드 트렌드 매핑
4. → 결과물: plan_trend-radar_YYYY-MM-DD.md
```

#### 🔍 STEP 3: 경쟁 환경 분석

**사용 스킬**: `/competitive` (경쟁 환경 분석)

무엇을 분석하는가:
- 직접 경쟁 (같은 타겟·같은 카테고리)
- 간접 경쟁 (다른 카테고리지만 같은 니즈 해결)
- 대안 (고객이 우리 대신 선택하는 것)

**와키윌리 사례 — TRUSTED COMFORT**:

| 포지션 | 경쟁 브랜드 | 고객 불만 (=우리 기회) |
|--------|-----------|---------------------|
| 프리미엄 웰니스 | 알로요가, 스포티앤리치 | "예쁜데 너무 비싸" (8~15만원대) |
| 국내 애슬레저 | 안다르, 젝시믹스 | "기능성은 좋은데 밖에 입기 부끄러워" |
| IP 굿즈 | 카카오프렌즈, 산리오 | "귀엽지만 옷이 아니라 굿즈" |
| 코지 SPA | 무인양품, 유니클로 | "편한데 예쁘지 않아" |
| **공백** | **없음** | **"귀엽고 힙하면서 편안한" = 와키윌리** |

→ **공백이 보이면 그것이 PMF 기회입니다.**

#### 🔍 STEP 4: 고객 리서치

**사용 스킬**: `/research-users` + `/interview`

무엇을 수행하는가:
- 페르소나 구체화 (UNI, WOMAN 페르소나 기반)
- 고객 인터뷰 가이드 설계
- 구매 여정 매핑

**고객 인터뷰 핵심 질문 (5가지)**:

| # | 질문 | 목적 |
|---|------|------|
| 1 | 최근에 산 옷 중 가장 마음에 드는 건? 왜? | 실제 구매 기준 파악 |
| 2 | 그 옷을 어떤 상황에서 입어? | 씬(Scene) 기반 니즈 발견 |
| 3 | 옷 살 때 가장 짜증나는 순간은? | Pain Point 발견 |
| 4 | 와키윌리 하면 떠오르는 이미지 3개는? | 브랜드 인식 확인 |
| 5 | 이 옷(컨셉보드 제시)에 얼마까지 낼 수 있어? | 가격 민감도 측정 |

### 3-3. STAGE 1 산출물 체크리스트

- [ ] 거시 환경 분석 완료 (시장 규모·성장률·트렌드)
- [ ] 코어 타겟 세그먼트 3개 이내로 구체화
- [ ] 각 세그먼트의 "상황(Scene)" 최소 3개 정의
- [ ] 경쟁 포지셔닝 맵 완성 (공백 식별)
- [ ] 고객 인터뷰 5명 이상 실시 (또는 VOC 데이터 분석)

## 4. STAGE 2 — 제품-가치 설계

> **핵심 질문**: "고객이 돈을 내는 진짜 이유가 뭔가?"

### 4-1. Job-to-be-Done (JTBD) — 고객이 우리 옷을 '고용'하는 이유

고객은 "옷을 산다"가 아니라 **특정 과업을 해결하기 위해 상품을 '고용'**합니다.

**와키윌리 사례 — TRUSTED COMFORT JTBD 분석**:

| 과업 (Job) | 상황 | 현재 해결 방식 | 현재 불만 (Pain) |
|-----------|------|-------------|----------------|
| 필라테스 → 카페 전환 없이 이동 | 수업 직후, 카페 약속 | 갈아입을 옷을 따로 챙김 | 안다르: "로고 보이면 부끄러워" |
| 산책·모닝워크용 착장 완성 | 아침 산책, 카페런 | 루즈핏 후드+조거 (무브랜드) | 전용 산책복이 없음 |
| 사우나 후 라운지 착장 | 사우나→카페 이동 | 무인양품 트레이닝복 | "편한데 예쁘지 않아" |
| 수면 루틴용 슬립웨어 세트 | 밤 루틴, 집순이 | 산리오 파자마 | "귀엽지만 옷이 아님" |

**핵심 인사이트**: 고객이 원하는 건 "옷 한 벌"이 아니라 **"갈아입지 않아도 되는 상태"**입니다.

### 4-2. FPOF 스킬로 가치 설계하기

#### 🎨 STEP 1: 밸류 프로포지션 설계

**사용 스킬**: `/value-prop` (밸류 프로포지션)

밸류 프로포지션 캔버스 구조:

```
┌─────────────────────────┐    ┌─────────────────────────┐
│     고객 프로필           │    │     가치 맵              │
│                         │    │                         │
│ Jobs: 고객의 과업         │◄──│ Products: 우리 상품       │
│ Pains: 불만·장애물       │◄──│ Pain Relievers: 해소제    │
│ Gains: 기대·혜택         │◄──│ Gain Creators: 촉진제     │
└─────────────────────────┘    └─────────────────────────┘
```

**와키윌리 사례 — TRUSTED COMFORT**:

| 고객 Pain | Pain Reliever (우리 해결책) |
|----------|--------------------------|
| 운동복은 밖에 입기 부끄럽다 | 200-280g 소프트 기능성 소재 — 운동복처럼 안 보이는 캐주얼 핏 |
| 예쁜 웰니스웨어는 너무 비싸다 | 알로요가 1/3 가격대 (3-5만원대) |
| IP 브랜드는 옷 퀄리티가 없다 | 패션 브랜드 품질 + IP 감성 |
| 씬(장면)마다 갈아입어야 한다 | "원씬 멀티 무드" — 한 착장으로 4가지 씬 커버 |

| 고객 Gain | Gain Creator (우리 강점) |
|----------|------------------------|
| SNS에 올릴 수 있는 센스 | 와키윌리 IP 포인트 디테일 |
| 우먼스 전용 핏 | UNI/W 이원화 — 우먼스 전용 실루엣 |
| 세트 구매의 만족감 | 캡슐 컬렉션 (상하의+액세서리 세트) |

#### 🎨 STEP 2: 가격 전략 설계

**사용 스킬**: `/pricing` (가격 전략)

**핵심 프레임워크 — 가격-가치 매트릭스**:

```
높은 가치
    │
    │  ★ 프리미엄    ○ 고가치
    │  (알로요가)     (와키윌리 목표)
    │
    │─────────────────────── 높은 가격
    │
    │  ▲ 경쟁가치     △ 저가치
    │  (유니클로)      (저가 SPA)
    │
낮은 가치
```

**와키윌리 가격 포지셔닝 원칙**:
- **Basic**: 접근성 (2-3만원대) — 재구매 유도
- **Trend**: 가치 중심 (3-5만원대) — "이 가격에 이 디자인"
- **Accent**: 감도 프리미엄 (5-8만원대) — 소량·한정

#### 🎨 STEP 3: 디자인 제너레이터

**사용 스킬**: `/design-generator` (AI 디자인 제너레이터)

가치 설계가 완료되면, 컨셉을 시각화합니다:
- 무드보드 생성
- 컬러 팔레트 제안 (`/color-intelligence` 연동)
- 실루엣·소재·디테일 방향 구체화

#### 🎨 STEP 4: 패턴 최적화 & 원가 설계

**사용 스킬**: `/pattern-optimizer` + 원가 검증

PMF에서 가격은 핵심 변수입니다. 가치 대비 가격이 맞아야 합니다.

| BTA | 원가 전략 | 목표 원가율 |
|-----|----------|-----------|
| Basic | 대량·안정 벤더·효율 소재 | 27-30% |
| Trend | 중기 리드타임·유연 리오더 | 30-33% |
| Accent | 소량·실험 소재·단기 생산 | 33-38% |

### 4-3. MVP (Minimum Viable Product) — 전부 만들지 말고 핵심만 테스트

PMF의 핵심은 **전체 라인업을 다 만들기 전에, 가장 자신 있는 2-3가지로 시장 반응을 검증**하는 것입니다.

**TRUSTED COMFORT MVP 설계 사례**:

```
전체 비전: 캡슐 4개(운동-외출-사우나-수면) × 다수 스타일
     ↓ (MVP로 축소)
MVP: 맨투맨+후드티 × UNI/W × 3색 = 12 STY
     ↓ (검증 후 확대)
시즌 론칭: 검증된 아이템 중심 확대 + QR 리오더
```

**MVP 선정 기준**:

| 기준 | 질문 | TRUSTED COMFORT 답변 |
|------|------|---------------------|
| 대표성 | 이 아이템이 라인업 전체를 설명하는가? | 맨투맨+후드티는 웰니스웨어의 코어 |
| 검증력 | 이 아이템 반응으로 전체를 예측할 수 있는가? | 이너웨어 반응 → 아우터/세트 확장 가능 판단 |
| 실행성 | 빠르게 만들 수 있는가? | 기존 생산 라인 활용 가능, 소재만 변경 |
| BTA 커버 | B/T/A 중 최소 2개를 포함하는가? | 베이직 컬러=B, 시즌 컬러=T |

### 4-4. STAGE 2 산출물 체크리스트

- [ ] JTBD 분석 완료 (최소 3개 핵심 과업 정의)
- [ ] 밸류 프로포지션 캔버스 작성 (Pain/Gain 매칭)
- [ ] 경쟁 가격 벤치마크 완료
- [ ] 목표 가격대 BTA별 설정
- [ ] MVP 라인업 확정 (12 STY 이내)
- [ ] 디자인 컨셉 무드보드 1차 완성

## 5. STAGE 3 — 채널-성장 검증

> **핵심 질문**: "이 상품이 고객에게 반복적으로 도달하고, 다시 사게 만들 수 있는가?"

### 5-1. 채널별 PMF 검증이 다르다

같은 상품이라도 채널에 따라 PMF 달성 여부가 다를 수 있습니다.

**와키윌리 채널 현황 및 PMF 검증 포인트**:

| 채널 | 25년 매출(백만) | 26년 목표 | 성장률 | PMF 검증 지표 |
|------|--------------|---------|-------|-------------|
| 샵인샵 | 32,074 | 35,502 | +11% | 정상 판매율, 회전율 |
| 직영점(FSS) | 7,053 | 10,058 | +43% | 객단가, 체류시간, 재방문율 |
| 자사몰 | 3,218 | 4,818 | +50% | 전환율, 장바구니 이탈률, 재구매율 |
| 온라인(외부몰) | 6,376 | 8,036 | +26% | 검색 전환율, 리뷰 점수, 찜 수 |
| 해외 | 843 | 6,896 | +718% | 현지 타겟 인지도, 첫구매 전환율 |

### 5-2. FPOF 스킬로 채널-성장 검증하기

#### 📊 STEP 1: 수요 예측 & 가격 최적화

**사용 스킬**: `/demand-optimizer` (수요 예측)

무엇을 수행하는가:
- 판매 데이터 기반 수요 예측
- 채널별 최적 가격 시뮬레이션
- QR(Quick Response) 리오더 타이밍 제안

**PMF 핵심 지표 — Sean Ellis Test (숀 엘리스 테스트)**:

고객에게 묻습니다:
> "이 상품이 더 이상 없다면 어떤 기분이 들겠어요?"

| 응답 | 비율 기준 | 의미 |
|------|---------|------|
| "매우 실망" | **40%+ 이면 PMF 달성** | 핵심 고객층 확보 |
| "약간 실망" | 합산 60%+ 양호 | 개선 여지 있으나 방향 맞음 |
| "상관 없음" | 50%+ 이면 PMF 미달 | 근본적 재설계 필요 |

**패션 버전 PMF 지표**:

| 지표 | PMF 미달 | PMF 달성 | 측정 방법 |
|------|---------|---------|----------|
| 정상 판매율 | < 50% | > 70% | 시즌 종료 시 정가 판매 비율 |
| 재구매율 | < 10% | > 25% | 6개월 내 재구매 고객 비율 |
| QR 리오더 | 0회 | 2회+ | 시즌 내 리오더 횟수 |
| SNS 자발 공유 | 광고만 | 고객 자발 | 브랜드 태그 UGC 비율 |
| NPS (추천 의향) | < 30 | > 50 | 순추천지수 |
| 리뷰 점수 | < 4.0 | > 4.5 | 무신사/자사몰 평균 |

#### 📊 STEP 2: 마케팅 전략 설계

**사용 스킬**: `/marketing` (마케팅 크리에이티브 툴킷)

PMF 검증을 위한 마케팅은 "많이 알리기"가 아니라 **"맞는 고객에게 맞는 메시지를 전달하고 반응을 측정"**하는 것입니다.

**B.A.M.P 검증 프레임워크** (전사 가이드라인):

| 단계 | 질문 | 검증 방법 |
|------|------|---------|
| **B**rand | 브랜드 메시지와 일치하는가? | 톤앤매너 체크 |
| **A**udience | 코어 타겟에게 도달하는가? | 타겟 도달률 측정 |
| **M**essage | 가치 제안이 명확한가? | A/B 테스트 |
| **P**erformance | 성과가 나오는가? | ROAS, 전환율 |

#### 📊 STEP 3: A/B 테스트 & 코호트 분석

**사용 스킬**: `/ab-test` + `/cohorts`

PMF를 "감"이 아닌 **데이터로 검증**합니다.

**A/B 테스트 사례 — 상품 상세페이지**:

| 변수 | A안 (기존) | B안 (PMF 최적화) |
|------|----------|----------------|
| 메인 이미지 | 모델 착용컷 1장 | 4씬 착용컷 (필라테스/카페/산책/라운지) |
| 카피 | "소프트 터치 스웻셔츠" | "필라테스에서 카페까지, 갈아입지 않아도 되는 스웻" |
| CTA | "구매하기" | "내 씬에 맞는 컬러 고르기" |
| 측정 | 전환율, 체류시간, 장바구니 추가율 | 동일 |

→ B안이 이긴다면, 고객은 "소재"가 아니라 **"씬 해결"**에 반응하는 것 → PMF 방향 확인

**코호트 분석 — 재구매 패턴**:

```
1월 첫구매 코호트 (100명)
├── 2월 재구매: 12명 (12%) → 아직 부족
├── 3월 재구매: 8명 (누적 20%) → 개선 중
└── 4월 재구매: 7명 (누적 27%) → PMF 근접

비교: 같은 시점 다른 라인 재구매율이 15%라면,
이 라인은 +12%p 더 강한 PMF 시그널을 보여줌
```

#### 📊 STEP 4: 그로스 전략 수립

**사용 스킬**: `/growth` (그로스 전략)

PMF가 확인되면 성장 가속화 단계로 전환합니다:

```
PMF 미달 상태: 상품/메시지 수정에 집중 (돈 쓰지 말 것)
    ↓
PMF 달성 초기: 채널 확대 + 리오더 확대
    ↓  
PMF 확산 단계: IMC 풀 캠페인 + QR 극대화 + 글로벌 테스트
```

**주의**: PMF 없이 마케팅 비용을 쓰면 "구멍 난 양동이에 물 붓기"입니다.

### 5-3. STAGE 3 산출물 체크리스트

- [ ] 채널별 PMF 지표 대시보드 설계
- [ ] Sean Ellis Test 또는 고객 설문 실시 (최소 50명)
- [ ] A/B 테스트 1건 이상 실행 및 결과 분석
- [ ] 코호트별 재구매율 추적 시작
- [ ] PMF 달성/미달 판정 기준 합의

## 6. BTA × PMF 연동 프레임워크

BTA 전략과 PMF는 별개가 아닙니다. **BTA 각 레벨에서 PMF의 의미가 다릅니다.**

### BTA별 PMF 정의

| BTA | PMF의 의미 | 핵심 지표 | 실패 시그널 |
|-----|----------|---------|-----------|
| **Basic** | "다시 산다" — 신뢰 기반 재구매 | 재구매율 > 30%, 회전율 | 재고 누적, 할인율 증가 |
| **Trend** | "갖고 싶다" — 욕구 기반 구매 | 정상 판매율 > 70%, SNS 반응 | 초두 미소진, 검색 저조 |
| **Accent** | "얘기하고 싶다" — 화제성 기반 확산 | UGC 수, 미디어 노출, 차시즌 전이율 | SNS 무반응, 전이 실패 |

### BTA × PMF 워크플로우

```
[Plan 단계]
┌──────────────┐
│ /trend-radar │ → 시장 기회 발견
└──────┬───────┘
       ↓
┌──────────────────┐
│ /competitive     │ → 경쟁 공백 식별
│ /research-users  │ → 고객 JTBD 발견
└──────┬───────────┘
       ↓
  "어디에 PMF 기회가 있는가?"
       ↓
[Design 단계]
┌──────────────────────┐
│ /value-prop          │ → Pain/Gain 매칭
│ /pricing             │ → BTA별 가격 설계
│ /design-generator    │ → 컨셉 시각화
│ /color-intelligence  │ → 컬러 전략
│ /pattern-optimizer   │ → 패턴·원가 최적화
└──────┬───────────────┘
       ↓
  "MVP 라인업 확정"
       ↓
[Do 단계]
┌──────────────────────┐
│ /marketing           │ → B.A.M.P 검증
│ /visual-factory      │ → 비주얼 콘텐츠
│ Techpack → 생산      │
└──────┬───────────────┘
       ↓
  "시장에 출시"
       ↓
[Check 단계]
┌──────────────────────┐
│ /demand-optimizer    │ → 수요/가격 분석
│ /ab-test             │ → 메시지 검증
│ /cohorts             │ → 재구매 분석
│ /metrics             │ → 대시보드
└──────┬───────────────┘
       ↓
  "PMF 달성 여부 판정"
       ↓
  ┌─── PMF 달성 ──→ /growth → 확대 (QR 리오더, 채널 확장)
  │
  └─── PMF 미달 ──→ [Act] 원인 분석 → Plan으로 돌아가 재설계
                     (Match Rate < 90% 시 자동 루프)
```

### PMF 판정 기준표

| 등급 | 조건 | 다음 행동 |
|------|------|---------|
| 🟢 **Strong PMF** | 정상판매율 70%+, 재구매 25%+, QR 2회+ | 풀 캠페인 + 라인 확장 |
| 🟡 **Emerging PMF** | 정상판매율 50-70%, 재구매 15-25% | 메시지/채널 튜닝 후 재검증 |
| 🔴 **No PMF** | 정상판매율 < 50%, 재구매 < 15% | 상품 or 타겟 재설계 (PDCA Act → Plan) |

### 핵심 용어 사전

| 용어 | 영문 | 설명 |
|------|------|------|
| PMF | Product-Market Fit | 상품이 시장 니즈와 맞는 상태 |
| JTBD | Jobs-to-be-Done | 고객이 상품을 사는 진짜 이유(과업) |
| MVP | Minimum Viable Product | 최소 검증 가능 상품 — 핵심만 먼저 테스트 |
| BTA | Basic-Trend-Accent | 상품·컬러 구성 전략 |
| QR | Quick Response | 시즌 내 빠른 리오더/스팟 생산 |
| NPS | Net Promoter Score | 순추천지수 (추천 의향 측정) |
| UGC | User Generated Content | 고객이 자발적으로 만드는 콘텐츠 |
| CAC | Customer Acquisition Cost | 고객 1명 획득 비용 |
| LTV | Lifetime Value | 고객 생애 가치 (평생 매출) |
| ROAS | Return on Ad Spend | 광고비 대비 매출 |
| 코호트 | Cohort | 같은 시점에 유입된 고객 집단 |
| 전이율 | Transition Rate | Accent 상품이 다음 시즌 Trend로 전환되는 비율 |

---

## 마무리: PMF는 한 번이 아니라 매 시즌 반복이다

```
PMF는 "한 번 달성하면 끝"이 아닙니다.

시즌마다 트렌드가 바뀌고,
고객의 니즈가 변하고,
경쟁 환경이 달라집니다.

매 시즌 PDCA를 돌며 PMF를 재검증하는 것,
그것이 FPOF 시스템의 핵심입니다.

Plan → 시장-고객 정의 (PMF 기회 탐색)
Design → 제품-가치 설계 (PMF 가설 수립)
Do → 시장 출시 (PMF 테스트)
Check → 데이터 검증 (PMF 판정)
Act → 개선 or 확대 (PMF 반복)
```

> **"예쁜 옷을 만드는 것이 아니라, 고객이 찾는 옷을 예쁘게 만드는 것."**
> — 이것이 패션에서의 Product-Market Fit입니다.
