# FPOF 퀵스타트 가이드

> 5분 안에 와키윌리 AI 패션 하우스를 시작하는 실전 가이드

---

## 이 시스템이 뭔가요?

AI가 와키윌리의 **6개 전문 팀, 20명의 전문가** 역할을 수행합니다.
당신이 자연어로 지시하면, 브랜드 DNA와 전략을 완벽히 숙지한 AI 전문가가 실무 산출물을 만들어 드립니다.

트렌드 분석부터 디자인, 테크팩, 마케팅, 매출 분석까지 — 시즌 전체를 함께 운영합니다.

---

## 시작하기

### 1단계: 대화 시작
Claude Code 또는 Codex에서 이 프로젝트 폴더를 열면 AI가 `CLAUDE.md`와 `.fpof-state.json`을 읽고 현재 상태를 파악합니다.

Codex에서는 아래 명령으로 동일한 운영 흐름을 사용할 수 있습니다.
```bash
make status      # 현재 상태 요약
make sync-state  # output 기준 상태 파일 동기화
```

### 2단계: 현재 상태 확인
```
"지금 어디까지 진행됐어?"
```
AI가 현재 시즌, PDCA 단계, 최근 작업 이력을 알려줍니다.

### 3단계: 작업 시작
하고 싶은 일을 자연어로 말하면 됩니다. AI가 알아서 적합한 전문가를 배정합니다.

---

## 첫 번째 시즌 기획 시작하기

### Plan 단계: "26SS 시즌 기획 시작하자"

**Step 1 — 트렌드 리서치**
```
"26SS 트렌드 분석해줘"
```
→ 시장 리서처가 글로벌 패션 트렌드, SNS 마이크로 트렌드, 경쟁사 동향을 분석합니다.

**Step 2 — 브랜드 전략**
```
"트렌드 기반으로 26SS 시즌 테마 제안해줘"
```
→ 브랜드 전략가가 와키윌리 DNA에 맞는 시즌 테마와 포지셔닝을 수립합니다.

**Step 3 — MD 전략**
```
"카테고리 믹스랑 챔피언 상품 전략 짜줘"
```
→ 수석 MD가 카테고리별 SKU 배분, 가격대, 챔피언 상품을 기획합니다.

**Step 4 — 라인시트**
```
"라인시트 만들어줘"
```
→ 컬렉션 플래너가 SKU별 상세, OTB, 사이즈/컬러 브레이크다운을 작성합니다.

**Step 5 — 검수 & 다음 단계**
```
"Plan 단계 검수해줘"
```
→ QG1을 실행하여 모든 산출물이 갖춰졌는지 확인합니다. PASS면 Design으로!

## 슬래시 명령어 (단축키)

자주 쓰는 작업을 `/명령어`로 빠르게 실행할 수 있습니다.

| 명령어 | 기능 |
|--------|------|
| `/status` | 현재 상태 한눈에 확인 |
| `/brief [유형]` | 산출물 문서 작성 (예: `/brief trend-brief`) |
| `/review` | 현재 단계 품질 검수 |
| `/next` | 다음 PDCA 단계로 전환 |
| `/team` | 팀 구성·역할 조회 |
| `/export` | 산출물 목록 정리 |
| `/deck [유형]` | 프레젠테이션(PPTX) 생성 (예: `/deck trend`) |
| `/pdf [유형]` | PDF 보고서 생성 (예: `/pdf techpack`) |
| `/sheet [유형]` | 엑셀(XLSX) 시트 생성 (예: `/sheet line-sheet`) |
| `/doc [유형]` | 워드(DOCX) 문서 생성 (예: `/doc campaign-plan`) |

## 알아두면 좋은 팁

### 구체적으로 말할수록 좋습니다
```
❌ "디자인 해줘"
✅ "키키 캐릭터를 활용한 유니섹스 그래픽 티셔츠 디자인해줘.
    타겟은 18~22세, 스트리트 감성으로"
```

### 특정 전문가를 지명할 수 있습니다
```
"패션 에디터한테 PDP 카피 맡길게"
"마케팅 디렉터가 GTM 계획 세워줘"
```

### 이전 산출물을 참조할 수 있습니다
```
"아까 만든 무드보드 기반으로 디자인 스펙 작성해줘"
"트렌드 브리프에서 뽑은 키워드로 시즌 테마 잡아줘"
```

### 승인은 당신이 합니다
AI는 절대 혼자 다음 단계로 넘어가지 않습니다. 매 단계마다 당신의 리뷰와 승인이 필요합니다.
```
"좋아, 이대로 진행하자" → 다음 단계로
"여기서 수정해줘: ..." → 수정 후 재확인
```

### 검수는 언제든 요청 가능합니다
```
"지금까지 한 거 검수해줘"
"Plan 산출물 빠진 거 없어?"
```

## 주의사항

1. **데이터는 직접 제공해야 합니다** — AI가 매출 데이터를 스스로 수집할 수 없습니다. Check 단계에서 실제 매출 데이터를 제공해주세요.

2. **AI 이미지는 참고용입니다** — 생성된 이미지는 방향성 확인용이며, 최종 제품 사진을 대체하지 않습니다.

3. **프리셋이 곧 브랜드입니다** — `presets/wacky-willy/` 폴더의 JSON 파일이 정확해야 모든 산출물의 품질이 보장됩니다.

4. **한 번에 하나씩** — 여러 작업을 동시에 요청하기보다, 하나씩 완료하고 확인하는 것이 품질이 높습니다.

5. **승인 없이 진행하지 않습니다** — AI는 반드시 계획 → 승인 → 실행 순서를 따릅니다. "알아서 해"는 금지입니다.
