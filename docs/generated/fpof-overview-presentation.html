PRINCIPLES

7대 핵심 원칙

  • 계획이 먼저 — "알아서 해"는 금지. 반드시 계획 → 승인 → 실행 순서
  • 브랜드 보이스 준수 — 고객 대면 콘텐츠는 반드시 tone-manner.json 참조
  • 한 번에 하나씩 — 큰 작업은 쪼개서 진행, 완료 후 상태 업데이트
  • 완료 전 셀프 체크 — 작업 완료 선언 전에 누락/오류 스스로 점검
  • 상태 기록 — 의미 있는 진전이 있으면 .fpof-state.json 업데이트
  • 참고자료가 진실 — 브랜드 정보를 지어내지 말 것. 프리셋 JSON 기반으로만
  • 토큰 비용 절감 — 에이전트 팀 운영 규칙 준수 (모델 믹싱, 최소 인원)

이중 트래킹

시즌 PDCA (전략 마일스톤) + 일상 운영 (주간 리뷰·회의록·대시보드)을 분리 관리. 일상 운영은 PDCA 단계와 무관하게 항상 실행 가능.

Quality Gate

각 단계 전환 시 자동 검수. 갭 분석 → 누락 항목 보완 → 완료 확인 후 다음 단계 진입.

상품 (P)

  • BTA 구성 필수 (Basic·Trend·Accent)
  • 3B 착장 금지
  • 로고 형태 변형 금지
  • SPA 보편성 + 스포츠 기능성 + 일본 감성

마케팅 (M)

  • B.A.M.P 기준 기획안 검증
  • IMC = 상품 중심
  • SNS 3균형 (뉴스/스타일링/로고)
  • 콜라보 연 2회 원칙

유통 (D)

  • VM = 소비자 관점
  • 보기 좋게 / 집기 좋게 / 사기 좋게
  • 아이캐칭은 메인/VP 집중
  • 동선/쇼핑 시간 절약 중심

📋

계획 승인 모드

팀원은 Plan Mode(읽기 전용)로 시작. 리드가 계획을 검토·승인한 후에만 구현 시작. 전사 가이드라인 미준수 시 자동 거절.

💰

모델 믹싱

리드 = Opus (판단·검토), 팀원 = Sonnet (실행·생성). 팀원 토큰 비용 4~5배 절감.

👥

최소 인원 원칙

팀원 3~5명 제한. 독립 작업은 서브에이전트 사용. 완료된 팀원 즉시 종료.

📡

브로드캐스트 최소화

전체 공지는 팀원 수만큼 토큰 소모. 개별 팀원에게 필요한 지시만 전달.
